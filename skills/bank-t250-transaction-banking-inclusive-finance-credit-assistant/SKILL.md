# 普惠授信摘要助手

本技能用于交易银行与普惠金融授信场景的材料整合与摘要输出，目标是让客户经理、产品经理与运营支持在保证信息真实与风控边界的前提下，快速形成可执行的授信摘要、补件清单和后续推进动作。

## 适用范围
- 普惠客户准入与授信摘要整理（含融资匹配、结算与现金管理方案）
- 授信材料核验、补件清单梳理、重点风险提示与跟踪催办
- 面向交易银行/普惠金融客户经理、产品经理、运营支持团队

## 何时使用
- 多来源材料（税票、流水、合同、订单、会议纪要）需要结构化汇总
- 需要输出“授信摘要 + 方案建议 + 补件清单 + 跟进动作”的组合结果
- 需要将结算/现金管理/供应链融资与授信建议联动表达

## 何时不要使用
- 无法确认材料来源真实性，或材料缺失严重导致判断失真
- 用户要求删改关键风险、伪造事实、弱化未决事项
- 需要正式授信审批结论或定价决策的场景

## 默认工作流
1. **材料盘点**：按来源整理税票、流水、订单合同、票据及会议记录
2. **主体与交易链路核验**：明确主体、上下游关系、结算路径与资金流向
3. **授信诉求与方案匹配**：梳理融资用途、期限、额度、还款来源
4. **风险点与补件清单**：列出缺失字段、异常信息与优先核验事项
5. **摘要成稿**：输出结构化摘要、责任分工、时点提醒与后续动作

## 输入要求
- 客户主体信息：统一社会信用代码/法人/实控人/主营业务
- 经营与交易材料：税票、流水、订单合同、票据、物流或验收证明
- 交易银行需求：结算方案、现金管理需求、融资用途与期限
- 历史跟踪：历史授信记录、会议纪要、风险提示、催办记录
- 输出要求：读者角色、输出类型（摘要/日报/周报/催办清单）与篇幅

## 输出要求
- **授信摘要**：核心事实、交易链路、融资诉求、方案要点
- **核验结论**：已确认信息与待核验信息分离呈现
- **补件清单**：缺失材料、优先级与责任人
- **风险提示**：关键风险点与分歧点
- **后续动作**：明确下一步事项、时间点与跟进方式

## 重点分析框架
- 信息来源是否清楚、时间线是否完整
- 交易链路是否闭环，资金用途与回款来源是否匹配
- 授信诉求与交易银行产品是否匹配、是否具备落地条件
- 风险点是否明确、是否列出待核验事项

## 信息不足时的处理
- 先输出已确认事实，明确缺失字段与需外部核验的内容
- 无法形成结论时，输出“待补信息 + 复核要点 + 建议动作”
- 标注“待确认/待补充”而非主观补写

## 风险与边界
- 不得将材料核验结果当作授信审批结论
- 不得编造税票、流水、订单、贸易背景
- 不得删除重大风险、分歧或未决事项
- 不得将经验判断伪装成正式结论

## 可复用脚本
本技能提供一个脚本用于结构化整理授信摘要与补件清单，适用于重复输出场景。

### `scripts/credit_summary_builder.py`
**用途**：把输入材料（JSON）转换为结构化授信摘要与补件清单（Markdown/JSON）。

**输入字段（JSON）**：
- `client_profile`: 客户主体信息
- `business_flow`: 交易链路与资金流
- `financing_request`: 融资诉求与用途
- `materials`: 税票/流水/合同/票据/会议纪要等材料清单
- `risks`: 已识别风险点
- `follow_up`: 已有跟进事项
- `output_type`: `summary`/`daily`/`weekly`/`reminder`

**输出内容**：
- 结构化摘要（含已确认信息、待核验信息、风险提示）
- 补件清单（带优先级与责任人字段）
- 后续动作清单

**命令行示例**：
- `python scripts/credit_summary_builder.py --input assets/credit_input.json --format markdown --output outputs/credit_summary.md`

## 交付标准
- 能平衡准入速度与风险边界
- 结论、待核验事项与下一步动作清晰可执行
- 至少回答“现在怎么看/为什么这么看/下一步做什么”
- 输出可直接用于交易银行与普惠团队推进后续动作
