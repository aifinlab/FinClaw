### 数据源
```bash
SCRIPTS="$SKILLS_ROOT/cn-stock-data/scripts"

# 日线 K 线（近 1 年，用于中长期分析）
python "$SCRIPTS/cn_stock_data.py" kline --code [CODE] --freq daily --start [1年前日期]

# 周线 K 线（近 3 年，用于长期趋势判断）
python "$SCRIPTS/cn_stock_data.py" kline --code [CODE] --freq weekly --start [3年前日期]

# 分钟线（短线分析）
python "$SCRIPTS/cn_stock_data.py" kline --code [CODE] --freq 60min --start [近期日期]

# 实时行情
python "$SCRIPTS/cn_stock_data.py" quote --code [CODE]

# 资金流向（验证量价信号）
python "$SCRIPTS/cn_stock_data.py" fund_flow --code [CODE] --days 30
```

### Workflow (6 steps):

**Step 1: 数据获取与预处理**
1. 获取多周期 K 线数据（日线+周线，短线加分钟线）
2. 获取实时行情（当前价、成交量、换手率）
3. 计算技术指标（在分析阶段手动计算）

**Step 2: 趋势分析**
基于日线和周线判断整体趋势：

**均线系统**（必算）：
- MA5 / MA10 / MA20 / MA60 / MA120 / MA250
- 均线排列：多头排列（短期>长期）/ 空头排列 / 缠绕
- 金叉/死叉：MA5 上穿/下穿 MA20
- 股价与均线关系：站上/跌破关键均线

**趋势判断**：
- 上升趋势：高点不断抬高 + 低点不断抬高
- 下降趋势：高点不断降低 + 低点不断降低
- 横盘震荡：在一定区间内波动

**Step 3: 量价分析**
- 量价齐升：健康上涨信号
- 量升价跌：可能是恐慌性抛售或底部放量
- 缩量上涨：上涨动力不足，警惕回调
- 缩量下跌：下跌动能衰减，可能接近底部
- 天量天价：可能是阶段性顶部信号
- 地量地价：可能是阶段性底部信号

关注换手率：
- < 1%：极度缩量，缺乏关注
- 1%-3%：正常换手
- 3%-7%：活跃交易
- 7%-15%：高度活跃，可能有重大变化
- > 15%：极端活跃，通常配合重大消息

**Step 4: 技术指标分析**

**MACD**（必算）：
- DIF = EMA12 - EMA26
- DEA = DIF 的 9 日 EMA
- MACD 柱 = 2 × (DIF - DEA)
- 信号：金叉（DIF上穿DEA）、死叉、零轴上方/下方、顶背离、底背离

**RSI**（相对强弱指标）：
- RSI(6) / RSI(14)
- > 80 超买区，< 20 超卖区
- 顶背离/底背离

**KDJ**：
- K/D/J 三线
- J > 100 超买，J < 0 超卖
- 金叉/死叉

**布林带（BOLL）**：
- 中轨 = MA20
- 上轨 = MA20 + 2×标准差
- 下轨 = MA20 - 2×标准差
- 触及上轨压力，触及下轨支撑，缩口后可能变盘

**Step 5: 支撑压力位与形态**

**支撑位识别**：
- 前期低点
- 重要均线（MA60/MA120/MA250）
- 成交密集区下沿
- 整数关口（心理价位）

**压力位识别**：
- 前期高点
- 重要均线
- 成交密集区上沿
- 套牢盘密集区

**常见形态**（如果数据中可识别）：
- 头肩顶/底、双顶/底、三角形收敛、旗形、楔形
- 突破形态的量能确认

**Step 6: 输出**

### 风格说明
| 维度 | formal（专业技术分析） | brief（个人看盘笔记） |
|------|---------------------|---------------------|
| 篇幅 | 2-4 页 | 半页 |
| 均线分析 | 完整（6 条均线 + 排列 + 金叉死叉） | MA20/MA60 关键位置 |
| 量价分析 | 详细（每日量价关系+资金流向验证） | 一句话量价判断 |
| 技术指标 | MACD+RSI+KDJ+BOLL 全面分析 | MACD 方向 + RSI 区间 |
| 支撑压力 | 多个层级 + 精确价位 | 最近的 1 个支撑 + 1 个压力 |
| 形态分析 | 完整形态识别+量能确认 | 一句话形态判断 |
| 结论 | 多空力量分析+关键价位 | 偏多/偏空 + 操作建议 |
| 免责声明 | 需要 | 不需要 |

### 关键规则
1. **多周期验证**：日线信号需要周线确认，分钟线服从日线
2. **量价配合**：任何价格信号都需要成交量确认
3. **技术分析的局限性**：明确说明技术分析无法预测突发事件（政策/黑天鹅），formal 模式必须提示
4. **不做精确预测**：给出关键价位区间而非精确数字（如"支撑区间 48-50 元"而非"支撑位 49.23 元"）
5. **结合资金面**：将 cn-stock-data 的资金流向数据与技术信号交叉验证
6. **A 股特色**：注意 T+1 交易制度、涨跌停板（10%/20% 创业板科创板）、集合竞价等因素对技术分析的影响
7. **技术指标需计算**：cn-stock-data 返回原始 OHLCV 数据，MA/MACD/RSI/KDJ/BOLL 等指标需要在分析时手动计算
