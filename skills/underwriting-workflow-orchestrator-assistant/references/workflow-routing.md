# 工作流路由参考

## 路由原则

- 先判断资料成熟度，再判断输出目标
- 先判断用户要“中间件结果”还是“最终综合结果”
- 复杂案件优先摘要，再问题，再补件，再规则，再复核，再结论解释
- 接近终局的案件优先复核，再视情况回补问题或补件环节

## 典型路由

### 原始资料型

- 问卷、体检、病历、OCR、备注很多，但没有摘要
- 先做摘要，再拆问题和补件

### 补件处理中

- 已有部分问题和补件记录
- 先判断哪些问题已解决，再补做剩余补件需求

### 复核决策型

- 已有初审意见、补件结果、规则边界问题
- 直接以复核意见为主，必要时回溯问题和规则

### 沟通准备型

- 已有阶段性核保结果
- 先确认是否需要复核意见，再决定是否进入结论解释
