# 零售客户存款流失预警助手

## 这个 skill 是做什么的
对零售客户存款余额与资金流向进行预警监测，识别异常流出、趋势性下滑与潜在流失风险，输出风险分层、驱动因素、处置建议与升级条件，支持客户经理与运营团队进行挽留与跟进。

## 适用范围
- 零售客户存款流失预警、客户经营与挽留
- 客户经理、零售运营、财富/存款经营团队
- 需要日报/周报级结构化输出

## 何时使用
- 需要持续监测存款余额与净流出趋势
- 需要给出可执行的触达/挽留建议

## 何时不要使用
- 仅凭单一异常就做违规定性
- 未提供历史基线或监测口径

## 默认工作流
1. 确认监测对象、时间窗口与历史基线
2. 计算净流出、余额跌幅与活跃度变化
3. 识别高风险信号与可能成因
4. 输出优先级、建议动作与升级路径

## 输入要求
- 余额时间序列（周/月）
- 近30/90天流入流出汇总
- 是否有工资/代发标记
- 客户等级、产品持有、触达记录
- 监测阈值与口径

## 输出要求
- 风险等级与优先级
- 关键驱动因素与证据
- 触达/挽留建议与升级条件
- 待核验事项与信息缺口

## 风险与边界
- 预警仅用于经营管理，不等同违规认定
- 必须标明时间窗口与阈值口径
- 数据缺失时需降级输出

## 信息不足时的处理
- 标注缺失字段
- 输出最小可行预警结论与补数建议

## 配套脚本
- `scripts/deposit_churn_warning.py`：计算余额趋势、净流出比例并输出风险分层。

### 脚本使用
```bash
python scripts/deposit_churn_warning.py --input deposit.json --rules rules.json --output out.json
```

### deposit.json 示例结构
```json
{
  "customer_id": "C1001",
  "balances": [
    {"date": "2026-01-01", "balance": 500000},
    {"date": "2026-02-01", "balance": 420000},
    {"date": "2026-03-01", "balance": 350000}
  ],
  "flows": {
    "inflow_30d": 60000,
    "outflow_30d": 180000
  }
}
```

### rules.json 示例结构
```json
{
  "thresholds": {
    "balance_drop_pct": 0.2,
    "outflow_ratio": 0.5
  }
}
```

## 交付标准
- 能说明“为什么预警”与“下一步怎么做”
- 输出可直接用于客户触达与跟进
