# 交易银行方案推荐方法说明

## 一、基本原则
方案推荐必须遵循“以需求为核心、以场景为基础、以落地为约束、以合规为底线”的方法。
推荐时不能只从产品角度出发，而应从企业经营、交易链路、组织架构和财务管理痛点出发。

## 二、标准步骤
1. 客户画像梳理
2. 经营与交易链路识别
3. 资金管理痛点归纳
4. 产品能力映射
5. 主方案与备选方案比较
6. 风险提示与实施建议输出

## 三、排序参考
- 需求匹配度高的优先
- 可快速落地的优先
- 对客户经营管理提升明显的优先
- 依赖条件少、改造成本可控的优先
- 合规边界清晰的优先

## 四、常见错误
- 只报产品名称，不讲为什么适合
- 忽略系统改造、数据接口和组织协同成本
- 未区分企业当前阶段需求和未来阶段需求
- 用理想状态替代现状分析
- 漏掉关键信息缺口说明
