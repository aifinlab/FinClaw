# 可疑交易数据初筛助手

## 1. 业务问题

交易流水量大、可疑信号分散，合规团队需要快速从海量流水中识别潜在可疑交易并确定复核优先级。本技能提供规则命中识别和初步筛查辅助，帮助缩小人工复核范围，但不替代正式可疑交易判定。

## 2. 适用场景

- 日常交易流水批量初筛，识别命中预设规则的交易记录
- 特定账户或客群的交易流水回溯筛查
- 合规团队复核前的优先级排序辅助
- 新规则上线前的历史数据回测验证
- 专项排查中的交易数据模式扫描

## 3. 输入

| 输入项 | 说明 | 是否必需 |
|--------|------|----------|
| 交易流水数据 | 含交易时间、金额、账户、对手方、渠道等字段 | 必需 |
| 筛查规则集 | 预设或自定义的筛查规则参数 | 可选（默认使用内置规则） |
| 账户基本信息 | 账户类型、开户时间、客户等级等 | 可选 |
| 历史筛查记录 | 过往命中及处置记录 | 可选 |
| 筛查时间范围 | 指定筛查的起止日期 | 必需 |

## 4. 输出

| 输出项 | 说明 |
|--------|------|
| 命中规则清单 | 每笔命中交易对应的规则编号及命中原因 |
| 复核优先级分层 | 高/中/低优先级分类及分层依据 |
| 命中统计摘要 | 按规则类别、账户、时间段的命中分布 |
| 模式特征描述 | 命中交易的资金结构、时间行为等特征概述 |
| 数据质量备注 | 缺失字段、异常值等数据质量问题标注 |

## 5. 默认分析维度

- **资金结构模式**：高频小额转入 + 集中大额转出、分散归集、化整为零
- **时间行为模式**：非常规时段交易（凌晨/节假日）、交易频率突增、周期性异常
- **对手方模式**：对手方高度集中、同名循环转账、短期内新增大量对手方
- **金额模式**：大额现金存取、金额临近报告阈值（如刚好低于5万元）、整数金额异常集中
- **渠道模式**：渠道与交易类型不匹配、频繁切换渠道

## 6. 默认分析框架

1. **数据预处理**：字段完整性检查，缺失值和异常值标注
2. **规则逐条匹配**：将交易流水与筛查规则集逐一比对，记录命中情况
3. **多规则叠加评估**：统计单笔/单账户的多规则叠加命中数量
4. **优先级评分**：根据命中规则数量、风险权重、金额规模综合评分
5. **分层输出**：按优先级分为高/中/低三层，高优先级优先呈现
6. **模式归纳**：对命中交易进行模式特征描述，辅助复核人员理解上下文
7. **结论先行**：先呈现命中数量和优先级分布，再展开明细

## 7. 安全边界

- 不编造数据；不夸大结论；缺失信息标注"未获取"或"待核实"
- 不替代正式审批、正式风控、正式合规、正式报送、正式处置、正式责任认定、正式管理决策
- 不对交易或客户做"疑似洗钱""疑似欺诈"等定性表述
- 仅提供数据模式分析和复核参考
- 不替代正式合规复核、审计、调查或账户处置决策
- 不替代可疑交易报告（STR）的正式判定
- 筛查结果仅为辅助参考，最终判定须由合规人员依据完整信息做出
