## 二、校验明细

### 校验1：两表合计笔数一致性

- 表1（按五级分类）合计笔数：**1000 笔**
- 表2（按担保方式）合计笔数：**1000 笔**
- 差异：0 笔

**结果：通过。**

### 校验2：两表合计余额一致性

- 表1（按五级分类）合计余额：**160,000.00 万元**
- 表2（按担保方式）合计余额：**160,500.00 万元**
- 差异：**500.00 万元**（表2 比表1 多 500.00 万元）

**结果：不通过。**
**严重性：阻断性。** 同一贷款总量在不同维度汇总后合计余额应完全一致，当前存在 500.00 万元差异，必须排查原因并修复。

### 校验3：表1 明细之和 vs 合计行

**笔数校验：**
- 明细之和：850 + 80 + 35 + 20 + 15 = **1,000 笔**
- 合计行：**1,000 笔**
- 结果：通过。

**余额校验：**
- 明细之和：125,000.00 + 18,000.00 + 8,500.00 + 5,200.00 + 3,300.00 = **160,000.00 万元**
- 合计行：**160,000.00 万元**
- 结果：通过。

### 校验4：表2 明细之和 vs 合计行

**笔数校验：**
- 明细之和：200 + 300 + 350 + 100 + 50 = **1,000 笔**
- 合计行：**1,000 笔**
- 结果：通过。

**余额校验：**
- 明细之和：28,000.00 + 45,000.00 + 62,000.00 + 15,000.00 + 10,500.00 = **160,500.00 万元**
- 合计行：**160,500.00 万元**
- 结果：通过。

### 校验5：表1 占比之和 vs 100%

- 明细占比之和：78.1% + 11.3% + 5.3% + 3.3% + 2.1% = **100.1%**
- 预期值：**100.0%**
- 差异：**+0.1 个百分点**

**结果：不通过。**
**严重性：阻断性。** 各分类占比之和应为 100.0%，当前为 100.1%，存在舍入累计误差或个别分项占比计算有误。

**复核明细（基于余额反算占比）：**

| 五级分类 | 贷款余额（万元） | 报表占比 | 反算占比（余额/合计） | 差异 |
|---------|----------------|---------|---------------------|------|
| 正常 | 125,000.00 | 78.1% | 78.125%（四舍五入 78.1%） | 无 |
| 关注 | 18,000.00 | 11.3% | 11.250%（四舍五入 11.3%） | **+0.05%** |
| 次级 | 8,500.00 | 5.3% | 5.3125%（四舍五入 5.3%） | 无 |
| 可疑 | 5,200.00 | 3.3% | 3.250%（四舍五入 3.3%） | **+0.05%** |
| 损失 | 3,300.00 | 2.1% | 2.0625%（四舍五入 2.1%） | **+0.04%** |

分析：关注类反算为 11.25%，四舍五入到一位应为 11.3%；可疑类反算为 3.25%，四舍五入到一位应为 3.3%；损失类反算为 2.0625%，四舍五入到一位应为 2.1%。多个分项在四舍五入时均向上取整，导致累计偏差 +0.1%。建议统一占比计算精度或对尾差项进行调整使合计为 100.0%。

### 校验6：表2 占比之和 vs 100%

- 明细占比之和：17.5% + 28.1% + 38.8% + 9.4% + 6.6% = **100.4%**
- 预期值：**100.0%**
- 差异：**+0.4 个百分点**

**结果：不通过。**
**严重性：阻断性。** 差异达 0.4 个百分点，超出正常舍入误差范围，需重点排查。

**复核明细（基于余额反算占比，使用表2合计 160,500.00 万元）：**

| 担保方式 | 贷款余额（万元） | 报表占比 | 反算占比（余额/合计） | 差异 |
|---------|----------------|---------|---------------------|------|
| 信用 | 28,000.00 | 17.5% | 17.445%（四舍五入 17.4%） | **报表多 0.1%** |
| 保证 | 45,000.00 | 28.1% | 28.037%（四舍五入 28.0%） | **报表多 0.1%** |
| 抵押 | 62,000.00 | 38.8% | 38.629%（四舍五入 38.6%） | **报表多 0.2%** |
| 质押 | 15,000.00 | 9.4% | 9.346%（四舍五入 9.3%） | **报表多 0.1%** |
| 组合 | 10,500.00 | 6.6% | 6.542%（四舍五入 6.5%） | **报表多 0.1%** |

分析：表2 各分项报表占比均高于反算占比，累计偏差 +0.4%（即 100.4% - 100.0%）。怀疑占比计算使用的分母并非表2合计值 160,500.00 万元，而可能使用了 160,000.00 万元（即表1合计值）作为分母。

**使用 160,000.00 万元作为分母重新反算：**

| 担保方式 | 贷款余额（万元） | 报表占比 | 以160,000为分母反算 | 差异 |
|---------|----------------|---------|-------------------|------|
| 信用 | 28,000.00 | 17.5% | 17.500% | 无 |
| 保证 | 45,000.00 | 28.1% | 28.125%（四舍五入 28.1%） | 无 |
| 抵押 | 62,000.00 | 38.8% | 38.750%（四舍五入 38.8%） | 无（或可为38.8%） |
| 质押 | 15,000.00 | 9.4% | 9.375%（四舍五入 9.4%） | 无 |
| 组合 | 10,500.00 | 6.6% | 6.5625%（四舍五入 6.6%） | 无 |

**关键发现：表2的占比是以 160,000.00 万元为分母计算的，而非表2自身合计值 160,500.00 万元。** 这进一步证实表2的余额合计 160,500.00 万元存在错误，正确合计应为 160,000.00 万元，与表1一致。表2某一分项或多项余额数据可能存在录入或加工错误。

## 四、修复优先级建议

1. **优先修复 P1**：排查表2余额差异 500.00 万元的来源。建议对比两表的底层明细数据，检查是否存在某笔/某批贷款在担保方式维度被重复计入或遗漏。
2. **修复 P2**：P1 修正后，以修正后的合计值为分母重新计算表2各项占比。
3. **修复 P3**：对表1占比进行尾差调整，使合计为 100.0%。
