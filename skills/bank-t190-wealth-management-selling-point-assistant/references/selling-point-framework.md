# 产品卖点框架（四段式）

## 结构

1. 卖点（客户可理解的价值点）
2. 证据（条款、策略、历史数据或公开口径）
3. 适配理由（与客户目标/约束的对应关系）
4. 风险边界（可能的波动、流动性限制、适当性要求）

## 常用卖点类型

- 收益路径：收益来源、分配机制、收益的可持续性解释
- 风险控制：回撤控制、波动管理、分散策略
- 流动性安排：开放周期、赎回规则、现金流匹配
- 目标匹配：与教育/养老/保值/增值/传承目标的适配
- 税务与合规：适用规则、禁止事项、必须披露事项

## 卖点表达要求

- 不使用收益保证性词语
- 一条卖点只讲一个核心价值
- 必须能对应到证据或条款
- 与客户目标和风险偏好强绑定
