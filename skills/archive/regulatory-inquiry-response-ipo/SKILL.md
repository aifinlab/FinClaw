# 监管问询回复助手（IPO 版）

## 触发场景

使用此技能当：
- 用户需要撰写 IPO 审核问询回复
- 用户要梳理问询问题的回复思路
- 用户需要组织问询回复的核查材料
- 用户要核查回复内容的完整性和准确性
- 用户提到"IPO 问询"、"审核问询"、"问询回复"、"反馈意见"等

## 核心工作流

### 1. 确认问询基本信息

首先向用户确认以下信息：
- **审核阶段**：交易所审核/证监会注册
- **问询轮次**：第一轮/第二轮/第三轮
- **问题数量**：共多少问题
- **回复截止日**：何时需要提交
- **用户角色**：保荐人/律师/会计师

### 2. 问询问题分类

将问询问题按以下类型分类：

#### 2.1 业务与技术类
- 商业模式与盈利模式
- 行业地位与竞争格局
- 核心技术来源与先进性
- 客户与供应商情况
- 业务合规性

#### 2.2 财务与会计类
- 收入确认政策
- 成本核算与毛利率
- 应收账款与存货
- 关联交易
- 财务内控

#### 2.3 法律与合规类
- 历史沿革与股权变更
- 资产权属
- 重大合同
- 诉讼仲裁
- 行政处罚

#### 2.4 募投项目类
- 募投项目必要性
- 投资估算合理性
- 经济效益测算
- 项目实施能力

#### 2.5 信息披露类
- 招股书披露完整性
- 风险因素披露
- 前瞻性信息

### 3. 回复撰写框架

为每个问题生成回复框架：

```markdown
## 问题 [X]：[问题标题]

### 问题原文
[交易所/证监会问询原文]

### 回复要点
1. [核心观点 1]
2. [核心观点 2]
3. [核心观点 3]

### 回复正文

#### 一、[分论点 1]
[详细论述 + 数据支撑]

#### 二、[分论点 2]
[详细论述 + 数据支撑]

#### 三、[分论点 3]
[详细论述 + 数据支撑]

### 核查程序
1. [核查程序 1]
2. [核查程序 2]
3. [核查程序 3]

### 核查结论
[明确结论]

### 附件材料
- [附件 1 名称]
- [附件 2 名称]
```

### 4. 回复撰写要点

#### 4.1 内容要求
- **针对性**：直接回应问题，不回避、不模糊
- **完整性**：覆盖问题所有方面，不遗漏
- **准确性**：数据准确、表述严谨
- **一致性**：与招股书披露一致，如有更新需说明

#### 4.2 论证方法
- **数据支撑**：用财务数据、业务数据说话
- **对比分析**：与同行业对比、与历史数据对比
- **逻辑清晰**：因果关系明确，论证链条完整
- **风险揭示**：如实披露风险，不隐瞒

#### 4.3 常见问题应对

| 问题类型 | 回复策略 |
|----------|----------|
| 质疑持续盈利能力 | 展示增长驱动、订单储备、市场空间 |
| 质疑财务真实性 | 提供核查程序、第三方证据、内控说明 |
| 质疑关联交易公允性 | 对比市场价、说明必要性、披露决策程序 |
| 质疑技术先进性 | 专利数量、研发投入、技术壁垒、行业认可 |
| 质疑合规性 | 取得合规证明、说明整改情况、律师意见 |

### 5. 核查材料组织

为每个问题梳理所需核查材料：

```markdown
## 问题 [X] 核查材料清单

### 内部文件
- [ ] 会议纪要
- [ ] 合同/协议
- [ ] 财务凭证
- [ ] 内部制度

### 外部文件
- [ ] 客户/供应商访谈记录
- [ ] 第三方报告
- [ ] 政府证明文件
- [ ] 资质证书

### 中介机构意见
- [ ] 保荐人核查意见
- [ ] 律师法律意见
- [ ] 会计师专项说明
```

### 6. 回复质量检查

提供回复质量检查清单：

- [ ] 问题是否全部回应
- [ ] 数据是否准确、来源是否注明
- [ ] 论证是否充分、逻辑是否清晰
- [ ] 与招股书披露是否一致
- [ ] 核查程序是否充分
- [ ] 附件材料是否完整
- [ ] 格式是否符合要求
- [ ] 是否存在敏感信息泄露

## 注意事项

- 回复应在规定时间内提交
- 重大问题应与监管预沟通
- 回复内容需经内部内核程序
- 注意保密，避免信息泄露
- 保留回复工作底稿
