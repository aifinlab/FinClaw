# 默认输出结构

除非用户另有要求，按以下顺序输出：

## 一、案件基本信息

- 案件号
- 产品/险种
- 门诊类型或就诊类型
- 出险/就诊时间
- 当前案件状态
- 材料来源或记录类型

## 二、责任判断摘要

- 用 3 到 5 条简明要点概括核心结论
- 优先说明是否初步可能属于责任范围、主要支持因素、主要限制因素和当前判断边界
- 如存在材料缺失、条款不清或争议较大，要在摘要中提示

## 三、门诊事实整理

- 就诊原因
- 门诊时间与次数
- 诊断信息
- 治疗或检查项目
- 医疗机构信息
- 费用类型与费用构成
- 已提交材料情况
- 其他与责任判断相关的重要事实

## 四、责任触发条件分析

- 当前案件对应的责任模块
- 门诊责任是否属于合同保障范围
- 是否满足责任启动条件
- 触发判断所依据的事实和条款点
- 如无法确认，应说明缺失的关键判断条件

## 五、赔付边界与限制因素

- 是否涉及免赔额
- 是否涉及赔付比例
- 是否涉及责任限额
- 是否涉及医保目录、自费项目或社保范围限制
- 是否涉及指定医院、合法合规医疗机构、合理必要治疗要求
- 是否涉及次数限制、项目限制、疾病范围限制或特殊门诊定义限制

## 六、免责与争议点分析

- 是否可能涉及等待期
- 是否可能涉及既往症或投保前已存在疾病
- 是否可能涉及责任免除情形
- 是否可能涉及非必要治疗、保健性质治疗、美容整形、非医学必需项目等常见争议因素
- 是否存在事实不清、材料不足、表述冲突或条款解释空间

## 七、材料支持度判断

- 当前材料是否足以支持初步判断
- 已具备的关键材料
- 缺失或待补充的关键材料
- 哪些材料缺失会直接影响责任结论

## 八、初步判断结论

仅使用以下之一：

- 可能属于责任范围
- 可能部分属于责任范围
- 可能不属于责任范围
- 暂无法判断

同时补充：

- 主要支持依据
- 主要限制依据
- 当前结论是否需要进一步复核

## 九、后续处理建议

- 是否建议补充材料
- 是否建议转人工复核
- 是否建议结合完整条款再次核验
- 是否建议进一步核实诊断、费用性质、就诊机构或等待期等关键事项
- 是否存在客户沟通风险或争议升级风险
