# 责任触发判断框架

按以下顺序做门诊责任判断：

## 1. 明确责任模块

- 先确认案件适用的是哪类门诊责任，例如普通门急诊、特定门诊、意外门诊、医疗险门诊费用、附加门诊责任。
- 若产品责任模块不清，先写明“责任模块待确认”，不要直接代入通用门诊责任。

## 2. 判断是否属于合同保障对象

- 门诊就医行为是否属于合同约定的保障场景。
- 就诊原因、诊断或费用项目是否落在责任描述可覆盖的范围。
- 若条款只保特定疾病、特定门诊类型、特定项目或特定就诊原因，要逐项对照。

## 3. 判断是否满足责任启动条件

- 是否已发生约定的保险事故或约定的门诊医疗行为。
- 是否在保险期间内发生。
- 是否满足等待期、起赔条件、次数条件、就医路径要求或其他前置条件。
- 是否有足够事实或材料支持上述条件。

## 4. 判断费用是否具备可赔基础

- 费用是否与本次门诊原因、诊断和治疗行为相关。
- 费用性质是否符合责任约定，例如药品费、检查费、治疗费、挂号费是否在保障范围内。
- 若费用清单混杂非相关项目、自费项目、保健项目或超范围项目，要单独提示。

## 5. 判断当前结论成熟度

- 条款清晰、事实清晰、材料充分时，可给出较明确的初步判断。
- 条款不全、责任模块不清、诊断不明、费用归属不清或材料不足时，优先写“暂无法判断”或“需补充核实”。
