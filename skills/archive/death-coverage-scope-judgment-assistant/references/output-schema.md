# 默认输出结构

除非用户另有要求，按以下顺序输出：

## 一、案件基本信息

- 案件号
- 产品/险种
- 身故类型
- 身故时间
- 当前案件状态
- 材料来源或记录类型

## 二、责任判断摘要

- 用 3 到 5 条简明要点概括核心结论
- 优先说明是否初步可能属于责任范围、主要支持因素、主要限制因素和当前判断边界
- 如存在材料缺失、条款不清、死亡原因不明或争议较大，要在摘要中提示

## 三、身故事实整理

- 身故时间与地点
- 身故前经过
- 发病、事故或抢救经过
- 死亡原因表述
- 医疗救治情况
- 涉及的事故或司法情况
- 已提交材料情况
- 其他与责任判断相关的重要事实

## 四、责任触发条件分析

- 当前案件对应的责任模块
- 身故责任是否属于合同保障范围
- 是否满足责任启动条件
- 触发判断所依据的事实和条款点
- 如无法确认，应说明缺失的关键判断条件

## 五、死亡原因与责任属性分析

- 当前身故更接近疾病身故、意外身故、猝死或其他特殊情形中的哪一类
- 支持该属性判断的核心依据
- 是否存在属性不清、原因待定或需进一步核实的情况
- 该属性对责任触发的主要影响

## 六、免责与争议点分析

- 是否可能涉及等待期
- 是否可能涉及自杀、自伤或故意行为
- 是否可能涉及违法犯罪、酒驾毒驾、高风险行为等责任免除情形
- 是否可能涉及既往症、投保前症状或已知重大健康异常
- 是否存在事实不清、材料不足、表述冲突或条款解释空间

## 七、材料支持度判断

- 当前材料是否足以支持初步判断
- 已具备的关键材料
- 缺失或待补充的关键材料
- 哪些材料缺失会直接影响责任结论

## 八、初步判断结论

仅使用以下之一：

- 可能属于责任范围
- 可能部分属于责任范围
- 可能不属于责任范围
- 暂无法判断

同时补充：

- 主要支持依据
- 主要限制依据
- 当前结论是否需要进一步复核或调查

## 九、后续处理建议

- 是否建议补充材料
- 是否建议转人工复核
- 是否建议转调查或法务进一步核实
- 是否建议结合完整条款再次核验
- 是否建议进一步核实死亡原因、事故性质、等待期、自杀条款适用或受益人资料等关键事项
- 是否存在客户沟通风险或争议升级风险
