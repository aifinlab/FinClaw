# 死亡原因与责任属性分析框架

## 1. 识别主要属性

- 疾病身故：病历、抢救记录、死亡证明、住院记录等更支持疾病进展导致身故。
- 意外身故：事故经过、现场材料、事故认定书、公安证明等更支持外来、突发、非本意、非疾病原因。
- 猝死：需要区分是否属于疾病性猝死、意外因素诱发或仍待明确原因。
- 其他特殊情形：自杀、自伤、刑事案件、工伤事故、交通事故、溺水、中毒等。

## 2. 提取支持依据

- 死亡证明或居民死亡医学证明（推断）书中的直接死因、根本死因和其他情况。
- 抢救记录、病历、出院记录或死亡记录中的诊疗经过。
- 公安、交警、法院、司法鉴定、尸检等外部文书中的原因认定。

## 3. 识别属性不清情形

- 死亡原因仅写“待查”“原因不明”或表述过于笼统。
- 医疗材料与事故材料对死因归因不一致。
- 事故发生后是否存在疾病因素介入、疾病发作后是否伴随事故因素，需要进一步分辨。

## 4. 输出规则

- 只写“更接近哪一类”或“目前倾向于哪一类”，不要在证据不足时写成最终定性。
- 对属性不清的案件，明确写“需进一步核实死亡原因或责任属性”。
