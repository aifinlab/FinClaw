# 问题优先级整理参考

## 必须优先追问

适合归入“必须优先追问”的情形：

- 当前关键风险无法判断，不确认就无法继续核保
- 已出现明确异常，但核心背景、时间线或结果不清
- 职业、收入、既有保单等关键变量缺失
- 资料前后冲突，必须先核实真伪或归属

## 建议补充追问

适合归入“建议补充追问”的情形：

- 当前已有初步判断基础，但补充后可明显提高判断完整性
- 属于边缘风险或背景信息，有助于减少后续反复追问
- 对复核、责任判断或定价有辅助价值

## 后续继续确认

适合归入“可在后续流程中继续确认”的情形：

- 不影响当前初步判断，但会影响完整留痕或后续复核
- 可与补件、面访或系统核查同步完成
- 现阶段缺口存在，但紧迫性相对较低

## 整理原则

- 优先保留最关键、最可执行的问题
- 能合并提问的，不拆成重复小问题
- 能通过内部系统核查的，不优先对外追问
- 边界不清时明确写“建议内部先核查”，不要直接定性为必须追问
