# 补件优先级整理参考

## 必须补件

适合归入“必须优先补充”的情形：

- 当前关键风险无法判断，缺资料会直接影响是否继续核保
- 已出现明确异常，但缺少核心原始报告、病历或说明
- 职业、收入、既有保单等关键核保变量缺失，无法继续评估
- 资料前后矛盾且必须核实，否则现有判断不可靠

## 建议补件

适合归入“建议补充”的情形：

- 当前已有初步判断基础，但补充后可明显提高判断完整性
- 属于边缘性风险或补充背景资料，有助于减少后续复补件
- 对责任范围、定价、观察期或复核意见有辅助判断价值

## 先内部核查

适合归入“需先内部核查后再决定是否补充”的情形：

- 现有系统、历史件、既有保单或内部备注中可能已有答案
- 补件范围是否需要扩大尚不明确
- 前端问题可能来自识别错误、归属错误或资料上传不完整

## 整理原则

- 优先收敛到最小必要补件集合
- 能通过内部系统确认的，不优先对外追件
- 能一次性说清范围的，不拆成多轮模糊追件
- 边界不清时明确写“建议内部先核查”，不要直接下成必须补件
