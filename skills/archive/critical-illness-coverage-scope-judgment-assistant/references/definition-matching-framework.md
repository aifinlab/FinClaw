# 重疾定义匹配框架

按以下顺序做重疾定义匹配：

## 1. 明确定义名称

- 先确认案件要对照的重疾名称或定义条款名称。
- 若存在轻症、中症、重疾同名或相近定义，要先确认对应层级，不要直接把疾病名称等同于重疾定义命中。

## 2. 拆解定义条件

- 疾病名称条件：是否为条款列明的疾病或同义表述。
- 诊断方式条件：是否要求病理、影像、专科医生诊断、实验室结果、手术记录或其他客观依据。
- 严重程度条件：是否要求达到特定分期分级、功能损害、侵袭程度、器官损伤或临床状态。
- 治疗或结果条件：是否要求实施特定手术、达到特定治疗阶段或形成特定后果。
- 时间或状态条件：是否要求持续一定时间、排除早期状态或满足首次确诊要求。

## 3. 对照案件事实

- 已明确满足的定义条件单独列示。
- 尚未明确、尚未满足或证据不足的定义条件单独列示。
- 对“接近满足但证据还不够”的情况，使用“可能部分匹配但需进一步核验”。

## 4. 识别常见边界

- 轻症、中症、重疾之间的层级边界。
- 恶性肿瘤早期与重度恶性肿瘤边界。
- 手术已建议但未实施、病理待回报、分期待明确、功能损害待观察等状态。
- 疾病名称已出现，但条款定义要求的严重程度、治疗条件或结果条件尚未满足。
