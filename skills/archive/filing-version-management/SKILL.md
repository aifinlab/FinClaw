# 申报材料版本管理助手

## 触发场景

使用此技能当：
- 用户需要跟踪申报材料版本变更
- 用户要对比不同版本的材料差异
- 用户需要生成版本更新说明
- 用户要确保申报材料版本一致性
- 用户提到"版本管理"、"版本对比"、"材料更新"、"申报稿版本"等

## 核心工作流

### 1. 确认版本管理基本信息

首先向用户确认以下信息：
- **项目类型**：IPO/再融资/并购/债券
- **材料类型**：招股书/募集说明书/报告书/法律意见书/审计报告
- **版本号**：[版本号，如 V1.0、V2.0]
- **版本日期**：[日期]
- **变更类型**：新增/修订/更新/更正

### 2. 版本命名规范

建立统一的版本命名规则：

```markdown
## 文件命名格式
[项目简称]_[文件类型]_[版本号]_[日期].[扩展名]

**示例：**
- XX 公司_招股说明书_V1.0_20240115.pdf
- XX 公司_法律意见书_V2.1_20240220.docx
- XX 公司_审计报告_V1.0_20240110.pdf

## 版本号规则
- V1.0：初稿
- V1.1、V1.2：小幅修订
- V2.0：重大修订
- V3.0：申报稿
- V3.1、V3.2：反馈回复更新
- V4.0：注册稿
```

### 4. 版本对比功能

#### 4.1 文本对比
- 使用 Word 比较功能或专业对比工具
- 标注新增、删除、修改内容
- 生成对比报告

#### 4.2 数据对比
- 财务数据一致性检查
- 业务数据一致性检查
- 前后文数据一致性检查

#### 4.3 格式对比
- 目录结构对比
- 页码连续性检查
- 格式规范检查

### 5. 版本一致性核查

确保各文件之间的一致性：

```markdown
## 一致性核查清单

### 财务数据一致性
- [ ] 招股书与审计报告数据一致
- [ ] 招股书与验资报告数据一致
- [ ] 招股书与评估报告数据一致
- [ ] 各文件财务数据勾稽关系正确

### 业务数据一致性
- [ ] 产能数据在各文件中一致
- [ ] 销量数据在各文件中一致
- [ ] 客户供应商数据在各文件中一致

### 法律事项一致性
- [ ] 历史沿革描述一致
- [ ] 股权结构描述一致
- [ ] 诉讼仲裁披露一致

### 信息披露一致性
- [ ] 风险因素披露一致
- [ ] 重大事项披露一致
- [ ] 前瞻性信息一致
```

### 6. 版本管理流程

#### 6.1 版本控制流程
1. 文件编制 → 2. 内部复核 → 3. 版本更新 → 4. 版本记录 → 5. 文件归档

#### 6.2 版本发布流程
1. 定稿确认 → 2. 版本编号 → 3. 发布通知 → 4. 旧版本归档 → 5. 新版本分发

## 注意事项

- 版本命名应统一规范
- 版本变更应及时记录
- 旧版本应妥善归档
- 确保各文件版本一致
- 注意保密，控制访问权限
- 定期备份版本文件
