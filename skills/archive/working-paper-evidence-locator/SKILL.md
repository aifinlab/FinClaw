# 底稿证据定位助手

## 触发场景

使用此技能当：
- 用户需要快速定位底稿中的支持性证据
- 用户要梳理核查程序与证据的对应关系
- 用户需要检查底稿完整性
- 用户要准备监管检查或内核检查
- 用户提到"底稿"、"证据"、"核查程序"、"工作底稿"等

## 核心工作流

### 1. 确认底稿类型

首先向用户确认底稿类型：
- **IPO 底稿**
- **再融资底稿**
- **并购底稿**
- **债券底稿**
- **持续督导底稿**

### 3. 证据定位功能

#### 3.1 按问题定位证据

当用户需要回应某个具体问题时，帮助定位相关证据：

```markdown
## 问题：[具体问题]

### 相关证据清单
| 序号 | 证据类型 | 证据名称 | 底稿编号 | 关键内容 |
|------|----------|----------|----------|----------|
| 1 | [类型] | [名称] | [编号] | [要点] |
| 2 | [类型] | [名称] | [编号] | [要点] |

### 证据摘要
[关键证据内容摘录]

### 证据链分析
[证据之间的逻辑关系]
```

#### 3.2 按核查程序定位证据

```markdown
## 核查程序：[程序名称]

### 程序要求
[监管/内核要求的核查程序]

### 已执行程序
- [程序 1] - [执行时间] - [执行人]
- [程序 2] - [执行时间] - [执行人]

### 支持证据
| 证据编号 | 证据名称 | 证明事项 |
|----------|----------|----------|
| [编号] | [名称] | [事项] |

### 程序充分性评价
[是否充分、是否需要补充]
```

### 4. 底稿完整性检查

提供底稿完整性检查清单：

```markdown
## 底稿完整性检查清单

### 必备文件
- [ ] 立项文件
- [ ] 尽调计划
- [ ] 尽调报告
- [ ] 内核意见
- [ ] 问询回复

### 业务底稿
- [ ] 访谈记录（高管/客户/供应商）
- [ ] 行业研究资料
- [ ] 业务合同
- [ ] 资质证照

### 财务底稿
- [ ] 审计报告
- [ ] 财务报表
- [ ] 银行流水
- [ ] 函证记录
- [ ] 凭证抽查

### 法律底稿
- [ ] 工商档案
- [ ] 资产权属文件
- [ ] 重大合同
- [ ] 诉讼仲裁文件
- [ ] 合规证明

### 缺失文件清单
| 序号 | 文件名称 | 缺失原因 | 补救措施 | 责任人 |
|------|----------|----------|----------|--------|
| 1 | [文件] | [原因] | [措施] | [姓名] |
```

### 5. 证据索引系统

建立证据索引便于快速定位：

```markdown
## 证据索引

### 按字母索引
- A - 资质证照
- B - 业务合同
- C - 财务数据
- F - 法律文件
- ...

### 按数字索引
- 001-100 - 业务尽调
- 101-200 - 财务尽调
- 201-300 - 法律尽调
- ...
```

### 6. 监管检查准备

为监管检查准备证据包：

```markdown
## 监管检查证据包

### 检查事项：[事项名称]

#### 证据目录
1. [证据 1] - [页码]
2. [证据 2] - [页码]
3. [证据 3] - [页码]

#### 证据说明
[每项证据的证明目的]

#### 核查结论
[基于证据的核查结论]
```

## 注意事项

- 底稿应及时整理、编号、归档
- 证据应清晰、完整、可追溯
- 注意保密，控制底稿查阅权限
- 电子底稿应备份
- 底稿保存期限应符合监管要求
