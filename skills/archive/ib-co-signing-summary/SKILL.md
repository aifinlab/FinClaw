# 投行会签摘要助手

## 触发场景

使用此技能当：
- 用户需要准备投行内部会签材料
- 用户要汇总各部门会签意见
- 用户需要跟踪会签进度和状态
- 用户要整理会签决议和后续行动
- 用户提到"会签"、"内部审批"、"立项会签"、"内核会签"等

## 核心工作流

### 1. 确认会签基本信息

首先向用户确认以下信息：

### 3. 会签类型差异化

#### 3.1 立项会签重点
- 项目来源和背景
- 客户资质和信誉
- 项目可行性和收益
- 团队配置和能力
- 利益冲突排查

#### 3.2 内核会签重点
- 尽调完成情况
- 主要问题和风险
- 材料完备性
- 是否符合申报条件
- 内核意见落实

#### 3.3 申报会签重点
- 申报文件齐备性
- 问询回复情况
- 重大事项更新
- 发行时机判断

### 4. 会签流程管理

#### 4.1 会签流程
1. 发起会签 → 2. 部门会签 → 3. 意见汇总 → 4. 项目组回复 → 5. 形成决议 → 6. 后续跟进

#### 4.2 会签时限
| 会签类型 | 会签时限 | 加急时限 |
|----------|----------|----------|
| 立项会签 | 5 个工作日 | 3 个工作日 |
| 内核会签 | 5 个工作日 | 3 个工作日 |
| 申报会签 | 3 个工作日 | 1 个工作日 |

### 5. 会签意见分类

| 意见类型 | 含义 | 处理方式 |
|----------|------|----------|
| 同意 | 无条件同意 | 直接进入下一环节 |
| 有条件同意 | 需满足特定条件 | 落实条件后确认 |
| 反对 | 不同意推进 | 需重新论证或终止 |
| 弃权 | 不发表意见 | 不计入表决 |

## 注意事项

- 会签材料应真实、准确、完整
- 会签意见应明确、具体
- 重大项目应组织会签会议
- 会签决议应形成书面记录
- 会签资料应妥善归档保存
- 注意保密，控制知悉范围
