# 监管问询回复助手（再融资版）

## 触发场景

使用此技能当：
- 用户需要撰写再融资审核问询回复
- 用户要梳理定增/配股/可转债问询问题的回复思路
- 用户需要组织再融资问询回复的核查材料
- 用户要核查回复内容的完整性和准确性
- 用户提到"定增问询"、"再融资问询"、"可转债问询"、"反馈意见"等

## 核心工作流

### 1. 确认问询基本信息

首先向用户确认以下信息：
- **融资方式**：定增/配股/可转债/其他
- **审核机构**：交易所/证监会
- **问询轮次**：第一轮/第二轮
- **问题数量**：共多少问题
- **回复截止日**：何时需要提交

### 2. 再融资问询问题分类

将问询问题按以下类型分类：

#### 2.1 融资必要性
- 前次募集资金使用情况
- 本次融资规模合理性
- 融资时机选择
- 替代融资方案分析

#### 2.2 募投项目
- 项目合规性（立项、环评、用地）
- 投资估算合理性
- 经济效益测算
- 项目实施能力
- 市场前景分析

#### 2.3 财务影响
- 对资产负债结构的影响
- 对每股收益的摊薄
- 对现金流的影响
- 财务可行性分析

#### 2.4 发行对象（定增）
- 发行对象资格
- 认购资金来源
- 锁定期安排
- 关联交易（如适用）

#### 2.5 转股条款（可转债）
- 转股价格确定
- 转股条款设计
- 赎回/回售条款
- 对原股东权益影响

#### 2.6 合规性
- 是否符合发行条件
- 是否存在禁止发行情形
- 信息披露合规性

### 3. 回复撰写框架

为每个问题生成回复框架：

```markdown
## 问题 [X]：[问题标题]

### 问题原文
[交易所/证监会问询原文]

### 回复要点
1. [核心观点 1]
2. [核心观点 2]
3. [核心观点 3]

### 回复正文

#### 一、[分论点 1]
[详细论述 + 数据支撑]

#### 二、[分论点 2]
[详细论述 + 数据支撑]

### 核查程序
1. [核查程序 1]
2. [核查程序 2]

### 核查结论
[明确结论]

### 附件材料
- [附件 1 名称]
- [附件 2 名称]
```

### 4. 再融资回复特殊要点

#### 4.1 融资必要性论证
- 展示资金缺口的测算过程
- 说明融资规模的合理性
- 分析股权融资 vs 债务融资的选择
- 论证融资时机的适当性

#### 4.2 募投项目论证
- 项目与公司战略的协同性
- 市场需求的充分论证
- 技术/人员/管理能力的匹配
- 风险因素的充分揭示

#### 4.3 摊薄影响分析
- 测算对即期回报的摊薄
- 说明填补回报的措施
- 控股股东/实控人承诺
- 董事/高管承诺

#### 4.4 前次募资使用
- 前次募集资金使用进度
- 使用效果与承诺对比
- 是否存在变更用途
- 是否存在违规行为

### 5. 常见问题回复策略

| 问题类型 | 回复策略 |
|----------|----------|
| 融资必要性不足 | 详细测算资金缺口、展示项目紧迫性 |
| 募投项目前景不明 | 提供市场调研、订单意向、可行性研究 |
| 摊薄影响较大 | 说明长期价值、填补回报措施 |
| 前次募资使用缓慢 | 解释原因、展示改进措施、调整计划 |
| 发行对象关联关系 | 披露关联关系、说明定价公允性 |

### 6. 核查材料组织

```markdown
## 问题 [X] 核查材料清单

### 募投项目相关文件
- [ ] 项目可行性研究报告
- [ ] 立项批复文件
- [ ] 环评批复
- [ ] 用地预审意见

### 财务测算文件
- [ ] 投资估算明细
- [ ] 效益测算过程
- [ ] 敏感性分析

### 合规性文件
- [ ] 董事会/股东大会决议
- [ ] 前次募资使用情况报告
- [ ] 填补回报措施承诺
```

### 7. 回复质量检查

- [ ] 融资必要性论证充分
- [ ] 募投项目可行性清晰
- [ ] 财务测算合理、假设审慎
- [ ] 摊薄影响分析完整
- [ ] 与前次披露一致
- [ ] 核查程序充分
- [ ] 附件材料完整
