# 门诊记录解析标准输出结构

本文件用于统一门诊记录解析输出结构，确保结果可直接用于核保审查、风险筛查和资料流转。

## 一、门诊基本信息

建议字段：

- 患者基本信息：
- 就诊时间：
- 医疗机构：
- 就诊科室：
- 材料来源或文本类型：

## 二、门诊记录解析结论摘要

建议输出 1 至 3 条：

- 是否存在明确诊断
- 是否存在反复就诊、持续治疗或活动性疾病
- 是否建议补问、补件或人工审核

## 三、主诉与诊断要点提取

重点覆盖：

- 主诉
- 现病史
- 初步诊断
- 待排查问题
- 本次就诊背景

## 四、就诊频次与病情活动性梳理

建议覆盖：

- 复诊频次
- 长期随访
- 症状持续或加重
- 多专科就诊

## 五、治疗措施与随访安排提取

建议按内容组织：

- 药物治疗
- 检查建议
- 转诊建议
- 观察方案
- 复查安排
- 当前治疗状态

## 六、核保关注点与风险提示

写法建议采用“已知事实 + 风险提示/判断边界”。

## 七、建议补充核实的问题或资料

问题应能直接用于业务人员、核保人员继续补问。

## 八、后续处理建议

可组合输出以下动作：

- 建议补充说明
- 建议补件
- 建议重点人工审核
- 建议结合问卷、病史、专项检查或住院资料进一步判断
- 当前可进入下一环节核保审查

必须避免写成正式核保结论。
