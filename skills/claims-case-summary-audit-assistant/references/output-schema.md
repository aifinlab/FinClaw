# 输出结构参考

当用户未指定格式时，按以下顺序组织输出：

## 一、案件基本信息
- 产品/险种
- 理赔申请类型
- 被保险人/申请人基本信息
- 当前理赔阶段
- 资料来源
- 材料来源或文本类型

## 二、案件摘要结论
- 用 3 到 5 条简明结论概括本次案件摘要结果
- 优先说明当前案件的核心事实、关键争议点、证据状态和待确认事项

## 三、案件事实与经过摘要
- 事故或发病经过
- 就医和治疗过程
- 关键时间点
- 申请事项与当前状态
- 如材料较完整，可按时间线整理重点事件

## 四、关键证据汇总
- 核心病历或诊断资料
- 事故或司法证明
- 关系或身份材料
- 费用或结算材料
- 其他关键支持性证据
- 对证据完整度不足的部分予以提示

## 五、责任争议点与审核难点
- 保险责任成立相关争议
- 免责边界相关争议
- 事故真实性、因果关系、定义适配、等待期、既往症、伤残等级、受益资格等问题
- 对每类争议尽量提炼最关键的判断点

## 六、材料完整性与一致性情况
- 已提供材料
- 缺失材料
- 已解决问题
- 未解决问题
- 前后矛盾点、真实性疑点和仍待核实事项

## 七、当前审核关注点与后续建议
- 当前最值得关注的审核重点
- 后续需重点核查的问题
- 建议补件、复核、升级审核、调查支持或进一步责任判断的方向
- 如资料较完整，也可说明“当前具备进入下一环节审核判断的基础”
