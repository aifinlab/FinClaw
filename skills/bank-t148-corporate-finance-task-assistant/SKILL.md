# 交易流水核验助手

本技能用于对对公交易流水做“可复核”的核验输出：先查样本覆盖与字段完整性，再查一致性（账户/对手/方向/余额/用途），最后识别异常特征并给出补件与升级复核建议。输出强调证据链与可追溯性。

## 适用范围
- 授信/尽调材料核验：流水真实性、回款逻辑、交易对手合理性
- 贷后排查：异常交易、资金挪用、第三方收付、回款拖延迹象
- 内部抽查：字段映射、导出范围、重复/缺失检查

## 何时使用
- 需要围绕流水核验做完整性、一致性、真实性或规则符合性检查时
- 需要把核验发现整理成异常项、补件清单和处理建议时

## 何时不要使用
- 用户要求绕过核验规则、伪造材料或淡化异常时
- 缺少最基本的原始材料或规则口径时

## 默认工作流
1. 定义核验边界：账户范围（是否全量）、时间窗口、样本方式（全量/抽样）
2. 完整性检查：字段缺失、日期不可解析、缺少对手/方向/金额
3. 一致性检查：重复导出、冲正/回退、跨期调账线索（若有）
4. 异常识别：大额整万、极端偏离、对手集中、第三方收付、用途异常
5. 输出：异常项清单（分级）+ 补件清单 + 升级复核建议

## 重点分析框架
- 材料是否齐备，字段是否缺失
- 同一事项在不同材料之间是否一致
- 是否存在伪造、篡改或逻辑冲突
- 核验结论会对流程推进造成什么影响

## 输入要求
- 最低可用输入：`company.name` + `verification.time_window` + `verification.transactions`
- 推荐输入见 `references/input-schema.md`

## 输出要求
- 结构化判断摘要
- 问题清单、待补资料和下一步推进建议
- 核验结论摘要
- 异常项清单、严重程度和建议动作
- 补件清单、复核建议和不可直接确认的事项

## 风险与边界
- 不得把初步分析包装成正式授信批复或最终审批结论
- 不得编造财务、行业或外部查询结果
- 不得把自动核验结果包装成最终认定
- 涉及造假、欺诈或监管敏感事项时要建议升级人工确认

## 信息不足时的处理
- 先列出已经掌握的事实，再明确缺失的关键字段，不要直接沉默
- 无法形成强结论时，优先输出框架、待补资料和优先核验事项
- 对依赖外部数据、制度文本或人工核实的部分，要单独标注[待补充/待确认]

## 交付标准
- 必须输出：样本覆盖说明 + 异常项（分级）+ 补件 + 升级复核建议

## 配套脚本（可选）

```bash
python scripts/run_skill.py --input assets/example-input.json --format markdown
python scripts/run_skill.py --input assets/example-input.json --format json
```

脚本入口：`scripts/run_skill.py`（调用 `shared/corporate_ops_skill_engine.py` 的 `t148` 场景）。
