# 客户陪伴框架说明

## 一、目标

客户陪伴的核心不是简单输出一句安抚话，而是在客户对产品、市场、风险、收益预期产生疑问或波动时，持续提供可理解、可执行、可追踪的信息支持，让客户维持对资产配置逻辑和持仓策略的理解。

## 二、典型陪伴目标

1. 降低因短期波动引发的焦虑
2. 减少因误解导致的非理性赎回
3. 提升客户对产品定位和风险收益特征的理解
4. 帮助客户接受分散配置、长期持有和定投等策略
5. 在关键节点保持服务温度和沟通连续性

## 三、核心结构

建议采用以下四段式：

1. 先确认客户关切
2. 再解释当前情况
3. 然后给出稳健动作建议
4. 最后约定后续跟进

## 四、常见客户场景

### 1. 净值回撤客户

重点：
- 解释回撤来源
- 说明产品定位
- 避免简单用“长期一定会回来”搪塞
- 强调观察点与后续跟踪

### 2. 收益不达预期客户

重点：
- 解释预期与实际差异的来源
- 说明市场环境变化
- 判断是否属于正常波动区间
- 提示是否需要再平衡或降低集中度

### 3. 情绪波动较大客户

重点：
- 降低刺激性表达
- 先共情再解释
- 优先提供简单清晰的框架
- 不宜一次性输出过多专业名词

### 4. 长期沉默客户

重点：
- 触达语气应轻量
- 内容应以关怀和提醒为主
- 避免过度推销

## 五、陪伴内容的评价标准

- 是否让客户更理解而不是更困惑
- 是否引导理性而不是制造情绪
- 是否有后续动作而不是停留在解释层面
- 是否在合规边界内表达
