# 住院资料解析标准输出结构

本文件用于统一住院资料解析输出结构，确保结果可直接用于核保审查、风险判断和资料流转。

## 一、住院基本信息

建议字段：

- 患者基本信息：
- 住院时间：
- 医疗机构：
- 入院科室：
- 材料来源或文本类型：

## 二、住院资料解析结论摘要

建议输出 1 至 3 条：

- 是否存在明确严重诊断
- 是否存在手术、并发症或恢复风险
- 是否建议补问、补件或人工审核

## 三、主要诊断与住院原因提取

重点覆盖：

- 入院诊断
- 出院诊断
- 伴随疾病
- 并发症
- 入院原因和症状

## 四、住院经过与诊疗时间线梳理

建议覆盖：

- 入院初始状态
- 关键检查
- 治疗过程
- 手术或特殊治疗
- 出院时状态

## 五、治疗措施与严重性线索识别

建议按风险主题组织：

- 药物治疗
- 手术或介入
- 并发症
- 危重线索
- 恢复不良风险

## 六、恢复情况与核保关注点

写法建议采用“已知事实 + 风险提示/判断边界”。

## 七、建议补充核实的问题或资料

问题应能直接用于业务人员、核保人员继续补问。

## 八、后续处理建议

可组合输出以下动作：

- 建议补充说明
- 建议补件
- 建议重点人工审核
- 建议结合问卷、病史、门诊随访或专项检查进一步判断
- 当前可进入下一环节核保审查

必须避免写成正式核保结论。
