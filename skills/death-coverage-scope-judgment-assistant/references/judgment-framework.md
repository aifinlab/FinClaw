# 责任触发条件框架

## 1. 明确责任模块

- 先确认案件适用的是疾病身故、意外身故、寿险身故、附加身故责任或其他特殊身故责任。
- 若责任模块不清，先写明“责任模块待确认”，不要直接代入通用身故责任。

## 2. 判断是否属于合同保障范围

- 当前身故情形是否属于合同约定的保障对象。
- 若产品区分疾病身故与意外身故，要先确认案件属性再判断是否命中对应责任。
- 若涉及年龄限制、附加责任条件、未成年人身故限制或返还责任，要单独提示。

## 3. 判断是否满足启动条件

- 是否在保险期间内发生。
- 是否满足等待期、身故责任启动条件或特定时间条件。
- 是否有足够事实和材料支持身故事实、死亡原因和责任属性判断。

## 4. 判断当前结论成熟度

- 条款清晰、死因清晰、材料充分时，可给出较明确的初步判断。
- 条款不全、死因待定、事故性质待查、司法材料未结或关键材料缺失时，优先写“暂无法判断”或“需补充核实”。
