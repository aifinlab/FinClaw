# IPO 材料初审助手 - 风险提示版

## 功能概述

本技能用于检查招股书"风险因素"章节的披露质量，确保风险揭示完整、充分、有针对性，符合监管要求。

## 适用场景

- 招股书风险章节撰写
- 风险披露质量自查
- 问询反馈风险补充
- 同行业风险对比分析

## 风险披露质量要求

### 1. 针对性

- ❌ 避免：泛泛而谈、模板化表述
- ✅ 要求：结合公司实际情况，量化分析

### 2. 充分性

- ❌ 避免：避重就轻、轻描淡写
- ✅ 要求：充分揭示可能的不利影响

### 3. 重要性

- ❌ 避免：罗列无关紧要的风险
- ✅ 要求：按重要性排序，突出重大风险

### 4. 可读性

- ❌ 避免：专业术语堆砌、冗长晦涩
- ✅ 要求：简明扼要、便于理解

## 使用流程

### 步骤 1：提取风险章节内容

将招股书风险因素章节提取为文本

### 步骤 2：运行检查脚本

```bash
cd "C:\Users\51994\Desktop\skills\证券 2\5. IPO 材料初审助手 - 风险提示版"
python scripts/check_risk_disclosure.py --input <风险章节文本> --output <检查结果路径> --industry <行业代码>
```

### 步骤 3：查看检查结果

检查报告将包含：
- 已披露的风险类型
- 缺失的风险类型
- 披露质量评估

### 步骤 4：补充完善

根据检查结果补充风险披露

## 输出示例

```markdown
## 风险因素披露检查报告

### 检查概况
- 检查日期：2026-03-15
- 应披露风险类别：9 类
- 已披露风险类别：7 类
- 风险条目数量：28 条

### 缺失风险类别
1. ⚠️ 技术风险 - 未披露技术迭代风险
2. ⚠️ 法律风险 - 未披露知识产权纠纷风险

### 披露质量评估

| 风险类别 | 披露质量 | 问题 |
|----------|----------|------|
| 市场风险 | 良好 | - |
| 经营风险 | 待改进 | 客户集中度风险缺少量化分析 |
| 财务风险 | 良好 | - |
| 技术风险 | 不足 | 缺少核心技术泄密风险 |

### 改进建议

1. **补充技术迭代风险**：说明行业技术发展趋势及公司应对能力
2. **量化客户集中风险**：披露前五大客户销售占比及变动趋势
3. **增加知识产权风险**：说明专利保护情况及潜在纠纷风险
```
