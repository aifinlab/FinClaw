# 技能说明

本技能用于支持基金业务中的宣介合规工作，重点解决「对基金宣传材料、路演稿、海报和渠道文案做合规风险扫描并输出修订建议。」这类任务。输出默认使用中文，要求结构清楚、证据边界明确，并能被投研、产品、销售、投顾或合规团队直接复用。

# 本任务重点关注的问题

- 材料里最容易引发误导或违规的点是什么
- 哪些问题必须立即整改
- 哪些内容需要升级到正式合规或法务复核
- 修订意见怎样写得具体且可执行

# 适用范围

- 宣传材料预审
- 敏感表述扫描
- 风险披露缺口检查
- 修订建议与升级事项整理

# 不适用范围

- 替代正式合规审批
- 缺少制度依据却直接判定通过
- 删除或淡化重大风险披露

# 输入要求

优先输入包括：

- 原始材料文本
- 适用渠道与客群
- 制度或合规要求
- 版本信息

可选增强输入包括：

- 历史整改意见
- 相关产品资料
- 敏感词库

当输入不完整时，必须先列出缺失材料，再基于现有信息输出阶段性判断，不得编造未确认事实。

# 核心分析框架

## 一、先确认材料版本和适用场景

- 明确材料版本、使用渠道、目标客群和审查范围。
- 先判断是预审、复审还是整改。
- 确认适用制度和风险等级。

## 二、逐段拆解敏感点

- 识别收益承诺、误导性比较、缺失披露和适当性问题。
- 先定位高风险，再处理中低风险。
- 修订建议要能直接落字。

## 三、保留正式复核边界

- 辅助审查不是最终审批。
- 所有结论都应写成建议修订或待复核。
- 制度依据不足时必须明确标注。

# 推荐工作流

1. 明确审查对象、版本、目标渠道和适用制度口径。
2. 拆解材料中的收益表述、风险披露、适当性表述和操作流程。
3. 标记高风险表述、缺失披露、措辞冲突和证据不足之处。
4. 输出可执行的修订意见、风险清单和升级处理建议。
5. 始终保留「需正式合规复核」的边界，不把辅助审查写成最终审批。

# 关键判断视角

- 材料版本、对客场景与制度要求
- 收益、风险、费用和流程表述是否完整一致
- 敏感措辞、误导性表达和缺失披露
- 修订优先级、升级事项和复核路径

# 输出要求

建议输出至少覆盖以下模块：

- 审查摘要
- 高风险问题清单
- 修订建议
- 升级事项
- 待核验制度依据

若任务涉及对客材料、话术或解释文案，必须同时输出「不能直接对外使用的前提说明」或「需正式复核」的提醒。

# 输出约束

- 不得写成「已通过」或「可直接外发」。
- 不得删除重大风险、费用和限制披露。
- 不得虚构制度依据或审批结论。

# 质量检查清单

- 是否先界定了任务边界、时间窗口、样本范围和使用场景。
- 是否把已确认事实、解释性判断和待核验事项分开表达。
- 是否给出了证据链，而不是只写结论标签。
- 是否保留了风险提示、限制说明和后续动作。
- 若内容可能进入对客或外部场景，是否保留了复核边界。

- 材料版本、对客场景与制度要求
- 收益、风险、费用和流程表述是否完整一致
- 敏感措辞、误导性表达和缺失披露
- 修订优先级、升级事项和复核路径
