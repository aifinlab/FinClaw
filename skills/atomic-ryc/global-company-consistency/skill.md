# Global Listed Company Consistency Check Skill

## Skill 名称
上市企业字段一致性校验（基于公网可查数据与法律法规）

## 数据来源
本 Skill 使用**公网可查**且可复核的数据来源，优先采用官方法规来源与公开证券信息来源。

### 1) 法律法规与规则来源（官方）
- 中国证监会：《上市公司信息披露管理办法》
- 中国证监会法规库：《中华人民共和国证券法》
- 上海证券交易所：《上海证券交易所股票上市规则（2025年4月修订）》
- 深圳证券交易所：《深圳证券交易所股票上市规则 / 创业板股票上市规则 / 规范运作》等公开规则

### 2) 上市公司公开信息来源（公网可查）
- 新浪财经 F10 / 公司资料页（公开网页）
- 交易所公告检索页（上交所/深交所公开披露页面）
- 用户可补充的公开年报 PDF / 招股书 / 公告 URL（可选）

> 说明：
> - 默认实现以**公开网页抓取与规则匹配**为主，不依赖私有数据库。
> - 如目标网站页面结构调整，解析规则需要同步更新。

## 功能
本 Skill 面向某一上市企业，完成“**法规检索 + 字段一致性校验 + 风险输出**”三类任务：

### A. 法规检索
根据“字段一致性/信息披露真实准确完整/重大变更披露”等主题，自动检索并整理适用规则，输出：
- 规则名称
- 发布/生效信息
- 规则链接
- 与字段校验相关的核心义务说明

### B. 企业字段采集
从公网公开页面采集企业核心字段，默认字段包括：
- 公司名称
- 股票代码
- 上市交易所
- 法定代表人
- 注册地址
- 办公地址
- 注册资本
- 成立日期
- 上市日期
- 所属行业

### C. 一致性校验
对多个公开来源中的同名字段进行归一化比对，输出：
- `一致`：字段值相同或标准化后等价
- `疑似不一致`：字段值存在明显冲突
- `信息缺失`：某一来源缺失字段
- `疑似未及时更新`：公告显示字段已变更，但资料页仍保留旧值

### D. 风险识别与解释
结合规则要求，对不一致情况给出解释性风险标签，例如：
- 低风险：仅格式差异（如“股份有限公司/股份公司”）
- 中风险：地址、法定代表人、注册资本等核心字段跨来源不一致
- 较高风险：近期公告显示已变更，但公开资料页未更新，存在信息同步/披露口径差异风险

### E. 结果输出
输出结构化 JSON / 控制台文本结果，包括：
- 企业基础信息
- 采集来源明细
- 字段比对结果
- 法规依据
- 风险摘要

## 使用示例
以下示例以命令行方式运行。

### 1) 查询法规关键词
```bash
python script/fetch_regulations.py --keyword 信息披露
```

### 2) 采集上市公司公开资料
```bash
python script/fetch_company_profile.py --symbol 600519
```

### 3) 对上市公司做字段一致性校验
```bash
python script/check_consistency.py --symbol 600519 --exchange SSE
```

### 4) 指定公告关键词辅助校验
```bash
python script/check_consistency.py --symbol 600519 --exchange SSE --announcement-keywords 法定代表人 注册地址 注册资本
```

### 5) 运行完整演示
```bash
python script/run_demo.py --symbol 600519 --exchange SSE
```

## 交易说明
### 适用场景
- 上市公司尽调前的公开信息快速核验
- 法务/合规/研究人员对字段口径进行初筛
- 对公告、资料页、规则之间的对应关系进行半自动检查

### 不适用场景
- 不能替代律师法律意见书
- 不能替代审计、核查底稿或工商穿透核验
- 不能直接证明违法违规，仅用于发现“疑点”和“需进一步核验”的线索

### 校验逻辑说明
- 优先比对公开资料页与近期公告中的核心字段
- 对中文地址、公司名称做基础归一化处理（去空格、全半角、常见后缀统一）
- 若近期公告标题或正文含“变更法定代表人/注册地址/注册资本”等关键词，则优先判定是否存在“资料页未同步更新”情况

### 风险提示
- 公网资料可能存在缓存、页面延迟更新、转载误差
- 交易所/资讯站页面结构变化会影响解析准确率
- 深交所、上交所、资讯网站对字段口径和更新时间可能并不完全一致
