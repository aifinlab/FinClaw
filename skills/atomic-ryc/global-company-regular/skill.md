# Global Listed Company Regulatory Rules Engine Skill

## 数据来源
本 Skill 使用**公网可查、优先官方来源**的数据，对某个上市企业进行规则引擎校验：

1. **国家法律法规数据库**：用于检索《公司法》《证券法》等上位法。
2. **中国证监会证券期货法规数据库**：用于检索上市公司监管、信息披露、减持、并购重组、处罚与监管措施等规则。
3. **上海证券交易所官网/规则体系**：用于检索股票上市规则、自律监管指引、纪律处分与监管问询。
4. **深圳证券交易所官网/规则体系**：用于检索股票上市规则、规范运作、监管函件与处分文件。
5. **巨潮资讯网（CNINFO）**：用于检索上市公司公告、定期报告、临时公告、问询回复与公开披露材料。

> 说明：脚本优先给出官方检索入口；对于可直接访问的公开接口，会尽量抓取少量结果用于自动化校验。若站点接口或参数调整，可在 `script/fetch_public_sources.py` 中单点修复。

## 功能
本 Skill 用于对**单个上市企业**执行“规则引擎校验”，输出结构化风控报告。默认支持以下能力：

1. **法规检索聚合**
   - 聚合国家法律法规数据库、证监会、上交所、深交所、巨潮资讯等公开来源；
   - 生成待核验法规与披露证据列表。

2. **公告与监管文本风险命中**
   - 扫描公开文本是否出现“立案、处罚、纪律处分、监管函、问询函、失信、被执行人”等高风险词；
   - 输出命中项并建议人工复核。

3. **规则引擎校验**
   - 内置规则包括：
     - 重大事项披露时效
     - 关联交易审批与披露
     - 董监高/股东减持合规
     - 高比例质押预警
     - 处罚/立案/监管函舆情命中
   - 根据企业事实输入（facts）与公网检索结果生成校验结论。

4. **结构化报告输出**
   - 输出 JSON 报告，包含：
     - 风险总览
     - 逐条规则结论
     - 事实输入
     - 公开证据链接

## 使用示例
### 1）安装依赖
```bash
pip install -r requirements.txt
```

### 2）直接运行
```bash
python script/main.py \
  --company "某上市公司" \
  --exchange "SSE" \
  --keywords 关联交易 立案 调查 \
  --facts script/sample_facts.json \
  --output report.json
```

### 3）示例输出说明
输出报告 `report.json` 中主要包含：
- `summary.total_rules`: 校验规则总数
- `summary.failed_rules`: 未通过规则数
- `summary.risk_level`: low / medium / high
- `results`: 每条规则的通过情况、风险等级、解释
- `evidence`: 对应的法规与公告来源

### 4）事实输入示例
`script/sample_facts.json` 可作为模板，支持扩展字段，例如：
```json
{
  "event_date": "2026-03-01",
  "disclosure_date": "2026-03-05",
  "materiality": "重大",
  "related_party_transaction": true,
  "transaction_amount": 15000000,
  "board_approved": false,
  "disclosed": true,
  "pledge_ratio": 62
}
```

## 交易说明
1. 本 Skill 仅用于**合规/风控/研究辅助**，不构成投资建议、法律意见或审计意见。
2. 规则引擎中的阈值（如披露时限、金额阈值、质押比例预警线）为**默认风控示例**，需要结合：
   - 最新有效法律法规
   - 企业上市板块
   - 企业章程/承诺事项
   - 内部合规制度
   做二次配置。
3. 公网数据可能存在延迟、页面改版、接口参数变化、标题歧义、同名主体误命中等问题，因此：
   - 自动结论仅作为初筛；
   - 对于高风险命中，必须人工核验主体、日期、事实、适用规则和时点有效性。
4. 对涉及重大交易、减持、并购重组、关联交易、处罚/立案等场景，建议配合法务、董办、券商、会计师或律师进行复核。
