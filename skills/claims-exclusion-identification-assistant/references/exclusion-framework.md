# 免责条款定位与分析框架

## 1. 先明确责任模块

- 先确认当前案件涉及的是哪一类责任模块，例如门诊、住院、重疾、身故、意外或其他附加责任。
- 不同责任模块下的免责条款适用边界可能不同，不要脱离责任模块孤立判断免责。

## 2. 先定位最相关免责条款类别

优先从以下方向定位：

- 时间类：等待期、观察期、既往症追溯期。
- 健康与病史类：既往症、投保前症状、先天性疾病、遗传性疾病、精神心理类因素。
- 行为类：自杀、自伤、故意行为、违法犯罪、酒驾毒驾、高风险运动或作业。
- 医疗行为类：非医学必要治疗、美容整形、保健治疗、实验性治疗、非约定医院或不合规就医。
- 特殊场景类：妊娠生育、职业风险、特定事故或特别约定除外。

## 3. 逐项建立事实与条款对应关系

- 条款要求了什么条件。
- 当前案件已经看到哪些支持性事实。
- 哪些条件仍未明确或证据不足。
- 条款与事实之间是否存在解释空间。

## 4. 输出规则

- 能定位但不能确认触发的，写“存在除外风险但需进一步核实”。
- 能明确看到不满足关键条件的，写“当前不支持触发除外责任”。
- 不把条款出现等同于免责成立。
