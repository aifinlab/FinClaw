# 默认输出结构

除非用户另有要求，按以下顺序输出：

## 一、案件基本信息

- 案件号
- 产品/险种
- 案件类型
- 关键出险/就诊/事故时间
- 当前案件状态
- 材料来源或记录类型

## 二、除外责任识别摘要

- 用 3 到 5 条简明要点概括核心结论
- 优先说明是否初步可能触发除外责任、主要支持因素、主要限制因素和当前判断边界
- 如存在材料缺失、条款不清、事实不明或争议较大，要在摘要中提示

## 三、案件事实整理

- 出险或发病经过
- 就诊、住院、治疗或事故处理经过
- 关键行为或状态信息
- 医疗或事故结果信息
- 已提交材料情况
- 其他与除外责任识别相关的重要事实

## 四、可能相关的除外责任条款

- 列出与本案可能相关的免责条款类别
- 每项条款的核心触发条件
- 当前案件与该条款的关联程度
- 如条款不完整，应说明当前仅基于已知条款进行识别

## 五、条款匹配分析

对每一项重点免责条款，分别说明：

- 条款名称或类别
- 关键触发条件
- 当前案件中已具备的匹配事实
- 当前案件中尚不明确或未满足的条件
- 初步匹配判断（可能匹配 / 部分匹配 / 当前不支持 / 暂无法判断）

## 六、争议点与待核实事项

- 哪些事实仍不清楚
- 哪些材料仍不足
- 哪些条款存在解释空间
- 哪些记录前后不一致
- 哪些点如进一步核实后可能影响免责判断方向

## 七、材料支持度判断

- 当前材料是否足以支持初步除外责任识别
- 已具备的关键材料
- 缺失或待补充的关键材料
- 哪些材料缺失会直接影响免责判断

## 八、初步识别结论

仅使用以下之一：

- 可能触发除外责任
- 存在除外风险但需进一步核实
- 当前不支持触发除外责任
- 暂无法判断

同时补充：

- 主要支持依据
- 主要限制依据
- 当前结论是否需要进一步复核或调查

## 九、后续处理建议

- 是否建议补充材料
- 是否建议转人工复核
- 是否建议转调查或法务进一步核实
- 是否建议结合完整条款再次核验
- 是否建议重点核实等待期、既往症、行为性质、事故归因、医学必要性等关键事项
- 是否存在客户沟通风险或争议升级风险
