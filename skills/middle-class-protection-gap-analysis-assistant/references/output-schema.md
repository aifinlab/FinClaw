# 中产客户保障分析报告输出结构

## 使用说明

- 默认按本文件顺序输出
- 若用户要求简版，可压缩每节内容，但不要改变结论逻辑
- 若用户要求详细版，可在各节补充“判断依据”“中产特征”和“待确认信息”
- 若用户要求面向客户沟通，可保持专业前提下适当口语化

## 标准结构

### 一、客户与家庭概况摘要

- 家庭成员结构
- 家庭当前阶段
- 收入与负债概况
- 当前家庭责任概况
- 当前保障规划重点

写法要求：

- 用 5 到 7 句话概括，不机械罗列原始资料
- 优先突出经济支柱、家庭责任、生活成本压力和当前分析重点
- 如资料不足，直接标注缺失边界

### 二、现有保障盘点

- 客户及家庭成员已有保障情况
- 已覆盖的主要风险责任
- 当前配置中的基础覆盖情况
- 当前保障结构中的明显不足或失衡点

写法要求：

- 可按成员或按险种归类，但最终要回到家庭整体保障结构
- 对保额、期限、缴费、社保、团险和储蓄型安排不清的部分，要标记不可确认
- 不要把“已有储蓄型保险”直接等同于“风险保障充分”

### 三、保障缺口分析

必须覆盖以下七项：

1. 家庭收入保障缺口
2. 重疾保障缺口
3. 医疗保障缺口
4. 意外保障缺口
5. 子女责任保障缺口
6. 养老或长期规划缺口
7. 资产安全与责任匹配方面的缺口

写法要求：

- 每一项都尽量写清楚“缺什么”“为什么缺”“影响是什么”
- 体现中产客户收入高、支出高、责任重、追求生活稳定性的特征
- 能做方向性判断时，不强行精确量化
- 能做成员拆分时，优先说明支柱、配偶、子女的差异

### 四、重点风险提示

至少输出 3 到 5 条重点风险，常见角度包括：

- 收入水平较高，但现金流安全保障不足
- 医疗与重疾保障层次错配
- 有房贷教育等责任，但身故责任保障偏弱
- 储蓄和养老安排先行，基础风险保障滞后
- 生活方式较高，但保障水平无法支撑家庭稳定性
- 家庭资产在积累，风险隔离和责任承接不足

写法要求：

- 风险提示要短、准、能直接用于客户沟通
- 不夸张，不使用恐吓式表达

### 五、保障配置优先级建议

- 第一优先级
- 第二优先级
- 第三优先级
- 优先级判断依据

写法要求：

- 优先级要与现金流安全、家庭责任和生活品质维持直接关联
- 建议写明“先补方向”，不默认写成“先买某产品”
- 若预算有限，必须体现取舍逻辑

### 六、后续方案方向

- 后续适合继续细化的保障方向
- 需要进一步确认的信息
- 可进入下一步产品匹配或方案设计的重点方向

写法要求：

- 明确下一步需要补充哪些关键资料
- 明确哪些方向可以进入产品匹配，但不要越过分析边界
- 涉及高风险金融安排时，必须提醒人工复核
