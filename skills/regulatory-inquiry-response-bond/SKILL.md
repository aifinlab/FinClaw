# 监管问询回复助手（债券版）

## 触发场景

使用此技能当：
- 用户需要撰写债券审核问询回复
- 用户要梳理公司债/企业债问询问题的回复思路
- 用户需要组织债券问询回复的核查材料
- 用户要核查回复内容的完整性和准确性
- 用户提到"债券问询"、"公司债问询"、"反馈回复"、"审核问询"等

## 核心工作流

### 1. 确认问询基本信息

首先向用户确认以下信息：
- **债券品种**：公司债/企业债/其他
- **审核机构**：交易所/交易商协会/发改委
- **问询轮次**：第一轮/第二轮
- **问题数量**：共多少问题
- **回复截止日**：何时需要提交

### 2. 债券问询问题分类

将问询问题按以下类型分类：

#### 2.1 发行人资质
- 是否符合发行条件
- 主体资质与信用状况
- 历史债券发行与兑付记录
- 是否存在禁止发行情形

#### 2.2 财务状况（核心）
- 资产负债结构与杠杆水平
- 盈利能力与稳定性
- 现金流状况
- 偿债指标分析
- 有息债务结构与到期分布
- 对外担保情况

#### 2.3 经营与业务
- 主营业务与商业模式
- 行业地位与竞争格局
- 经营稳定性与可持续性
- 重大不利变化

#### 2.4 募投项目（如有）
- 项目合规性
- 投资估算与资金需求
- 经济效益测算
- 项目实施风险

#### 2.5 增信措施（如有）
- 担保人资质与代偿能力
- 抵质押资产权属与价值
- 增信措施有效性

#### 2.6 偿债保障
- 偿债资金来源
- 偿债计划安排
- 应急偿债措施
- 投资者保护条款

### 3. 回复撰写框架

为每个问题生成回复框架：

```markdown
## 问题 [X]：[问题标题]

### 问题原文
[审核机构问询原文]

### 回复要点
1. [核心观点 1]
2. [核心观点 2]
3. [核心观点 3]

### 回复正文

#### 一、[分论点 1]
[详细论述 + 数据支撑]

#### 二、[分论点 2]
[详细论述 + 数据支撑]

### 核查程序
1. [核查程序 1]
2. [核查程序 2]

### 核查结论
[明确结论]

### 附件材料
- [附件 1 名称]
- [附件 2 名称]
```

### 4. 债券回复特殊要点

#### 4.1 偿债能力论证（核心）
- **短期偿债能力**：流动比率、速动比率、货币资金覆盖
- **长期偿债能力**：资产负债率、利息保障倍数
- **现金流覆盖**：经营现金流对债务的覆盖
- **融资能力**：银行授信、再融资空间

#### 4.2 信用风险揭示
- 如实披露信用风险因素
- 说明风险缓释措施
- 不回避敏感问题
- 与评级报告保持一致

#### 4.3 财务数据核查
- 财务数据真实性核查
- 异常指标解释
- 与同行业对比
- 与历史数据对比

#### 4.4 特殊发行人类型

**城投类发行人：**
- 政府支持情况说明
- 隐性债务排查
- 区域财政状况
- 平台定位与转型

**房地产发行人：**
- 土储情况与去化
- "三道红线"指标
- 融资渠道与成本
- 销售回款情况

### 5. 常见问题回复策略

| 问题类型 | 回复策略 |
|----------|----------|
| 资产负债率偏高 | 说明行业特点、展示现金流覆盖、融资空间 |
| 盈利能力下滑 | 分析原因、展示改善措施、行业对比 |
| 短期债务集中到期 | 展示偿债安排、授信储备、再融资计划 |
| 对外担保较多 | 披露被担保方资质、反担保措施、代偿风险 |
| 募集资金用途不明 | 细化资金使用计划、说明必要性、披露管理制度 |

### 6. 核查材料组织

```markdown
## 问题 [X] 核查材料清单

### 财务核查
- [ ] 审计报告
- [ ] 财务报表明细
- [ ] 银行流水
- [ ] 征信报告

### 业务核查
- [ ] 主要合同
- [ ] 客户/供应商访谈
- [ ] 行业数据

### 合规核查
- [ ] 征信查询
- [ ] 诉讼查询
- [ ] 行政处罚查询
- [ ] 合规证明
```

### 7. 回复质量检查

- [ ] 偿债能力论证充分
- [ ] 财务数据真实准确
- [ ] 风险揭示充分
- [ ] 与募集说明书一致
- [ ] 核查程序充分
- [ ] 附件材料完整
- [ ] 符合监管规则要求

## 注意事项

- 回复应突出偿债能力分析
- 财务数据应覆盖最近三年及一期
- 关注表外负债和隐性债务
- 回复内容需经内部内核程序
- 保留核查工作底稿
