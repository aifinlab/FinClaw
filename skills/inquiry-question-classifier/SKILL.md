# 问询问题分类助手

## 触发场景

使用此技能当：
- 用户需要快速识别问询问题的类型
- 用户要确定问询问题的回复责任方
- 用户需要梳理问询问题的优先级
- 用户要统计问询问题的分布情况
- 用户提到"问题分类"、"问询类型"、"责任分工"、"问题统计"等

## 核心工作流

### 1. 问题分类体系

建立多层级问题分类体系：

#### 一级分类（按专业领域）
- **业务类** - 商业模式、市场、竞争、技术等
- **财务类** - 收入、成本、利润、资产、负债等
- **法律类** - 股权、资产、合同、诉讼、合规等
- **募投类** - 项目必要性、可行性、效益等
- **其他类** - 信息披露、发行方案等

#### 二级分类（按具体事项）

| 一级分类 | 二级分类 | 典型问题关键词 |
|----------|----------|----------------|
| 业务类 | 商业模式 | 盈利模式、收入来源、客户 |
| 业务类 | 市场竞争 | 市场份额、竞争对手、行业地位 |
| 业务类 | 技术实力 | 核心技术、专利、研发 |
| 业务类 | 业务合规 | 资质、许可、处罚 |
| 财务类 | 收入确认 | 收入、确认、时点、方法 |
| 财务类 | 成本核算 | 成本、毛利率、结转 |
| 财务类 | 资产质量 | 应收、存货、减值 |
| 财务类 | 关联交易 | 关联方、公允性、依赖 |
| 法律类 | 历史沿革 | 股权、变更、出资 |
| 法律类 | 资产权属 | 房产、土地、专利、商标 |
| 法律类 | 重大合同 | 合同、履约、风险 |
| 法律类 | 诉讼仲裁 | 诉讼、仲裁、纠纷 |
| 募投类 | 必要性 | 必要性、产能、市场 |
| 募投类 | 可行性 | 技术、人员、实施 |
| 募投类 | 效益测算 | 收益、回报、测算 |

### 2. 问题分类流程

对每个问询问题进行分类：

```markdown
## 问题 [X] 分类

### 问题原文
[问题全文]

### 分类结果
- **一级分类**：[业务类/财务类/法律类/募投类/其他]
- **二级分类**：[具体分类]
- **问题标签**：[标签 1]、[标签 2]、[标签 3]

### 分类依据
[说明分类理由]

### 复杂问题说明
[如问题涉及多个领域，说明主要关注点]
```

### 3. 责任方分配

根据问题分类，确定回复责任方：

| 问题类型 | 主责方 | 配合方 |
|----------|--------|--------|
| 业务类 | 保荐人 | 发行人 |
| 财务类 | 会计师 | 保荐人、发行人 |
| 法律类 | 律师 | 保荐人、发行人 |
| 募投类 | 保荐人 | 发行人、可研机构 |
| 行业数据 | 保荐人 | 第三方机构 |
| 综合类 | 保荐人 | 律师、会计师、发行人 |

### 4. 优先级评估

根据问题性质评估优先级：

#### 4.1 优先级标准

| 优先级 | 标准 | 响应时间 |
|--------|------|----------|
| **高** | 涉及发行条件、重大风险、监管关注重点 | 24 小时内 |
| **中** | 重要事项、需详细论证 | 48 小时内 |
| **低** | 一般事项、补充说明 | 72 小时内 |

#### 4.2 高优先级问题特征
- 质疑持续盈利能力
- 质疑财务真实性
- 质疑合规性
- 质疑独立性
- 重大信息披露遗漏

### 6. 问题关联分析

识别问题之间的关联性：

```markdown
## 问题关联分析

### 关联问题组
**主题**：[主题名称]
- 问题 1：[问题编号] - [问题简述]
- 问题 2：[问题编号] - [问题简述]
- 问题 3：[问题编号] - [问题简述]

### 回复建议
[建议统一回复或相互呼应]
```

## 注意事项

- 分类应准确反映问题实质
- 复杂问题可能涉及多个领域，需明确主责
- 优先级评估应考虑监管关注程度
- 关联问题应协调回复口径
- 分类结果应及时同步各中介机构
