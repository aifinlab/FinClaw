# 问询材料引用助手

## 触发场景

使用此技能当：
- 用户需要在问询回复中引用底稿证据
- 用户要规范引用格式确保统一
- 用户需要确保引用材料来源可追溯
- 用户要检查引用的完整性和准确性
- 用户提到"引用"、"附件"、"证据"、"核查材料"等

## 核心工作流

### 1. 引用材料分类

按材料类型建立引用规范：

#### 1.1 内部文件引用
| 材料类型 | 引用格式 | 示例 |
|----------|----------|------|
| 访谈记录 | 访谈纪要 [编号] | 访谈纪要 BP-005 |
| 会议纪要 | 会议纪要 [日期] | 会议纪要 2024-01-15 |
| 内部报告 | 报告名称 [版本] | 尽调报告 V2.0 |
| 核查记录 | 核查表 [编号] | 核查表 FP-023 |

#### 1.2 外部文件引用
| 材料类型 | 引用格式 | 示例 |
|----------|----------|------|
| 审计报告 | 事务所 + 报告号 | 天健审 [2024]001 号 |
| 法律意见 | 律所 + 文号 | 国浩律师 [2024]001 号 |
| 评估报告 | 机构 + 报告号 | 中企华评报字 [2024]001 号 |
| 行业报告 | 机构 + 报告名 | 弗若斯特《行业报告》 |

#### 1.3 官方文件引用
| 材料类型 | 引用格式 | 示例 |
|----------|----------|------|
| 营业执照 | 公司 + 证照类型 | 发行人营业执照 |
| 资质证书 | 证书名称 + 编号 | 高新技术企业证书 GR2023XXX |
| 批复文件 | 机关 + 文号 | 证监会证监许可 [2024]X 号 |
| 合规证明 | 机关 + 日期 | XX 局合规证明 2024-01 |

#### 1.4 合同协议引用
| 材料类型 | 引用格式 | 示例 |
|----------|----------|------|
| 业务合同 | 合同名 + 对方 + 日期 | 与 A 公司销售合同 2023-01 |
| 借款合同 | 银行 + 合同号 | XX 银行借款合同 XXX 号 |
| 担保合同 | 担保方 + 日期 | XX 公司担保合同 2023-06 |

### 2. 引用格式规范

#### 2.1 正文引用格式

```markdown
### 规范格式

根据 [材料名称]（[引用编号]），[内容说明]。

**示例：**
根据天健会计师事务所出具的《审计报告》（天健审 [2024]001 号），发行人 2023 年度营业收入为 XX 万元。

根据对发行人总经理 XX 的访谈记录（访谈纪要 BP-005），发行人主要采用直销模式开拓市场。
```

#### 2.2 附件引用格式

```markdown
### 附件列表格式

**附件 X-X：[附件名称]**
- 来源：[材料来源]
- 日期：[文件日期]
- 页码：[关键页码]
- 证明事项：[证明目的]

**示例：**
**附件 3-1：发行人前十大客户销售合同**
- 来源：发行人提供
- 日期：2023 年度
- 页码：P1-50
- 证明事项：发行人与主要客户的合作关系及交易条款
```

### 3. 引用索引系统

建立引用索引便于追溯：

```markdown
## 引用索引

### 按回复问题索引
| 问题编号 | 引用材料编号 | 材料名称 | 页码 |
|----------|--------------|----------|------|
| 问题 1 | 附件 1-1 | 审计报告 | P1-100 |
| 问题 1 | 附件 1-2 | 访谈纪要 | P5-10 |
| 问题 2 | 附件 2-1 | 法律意见 | P20-30 |

### 按材料类型索引
| 材料类型 | 编号范围 | 数量 |
|----------|----------|------|
| 财务类附件 | 附件 1-1 ~ 1-50 | 50 |
| 法律类附件 | 附件 2-1 ~ 2-30 | 30 |
| 业务类附件 | 附件 3-1 ~ 3-40 | 40 |
```

### 4. 引用完整性检查

提供引用检查清单：

```markdown
## 引用完整性检查

### 检查项目
- [ ] 所有引用材料均有明确来源
- [ ] 引用格式统一规范
- [ ] 附件编号连续无遗漏
- [ ] 正文引用与附件列表对应
- [ ] 关键数据均有出处
- [ ] 引用材料在有效期内
- [ ] 涉密信息已脱敏处理

### 常见问题
| 问题类型 | 示例 | 修正方法 |
|----------|------|----------|
| 来源不明 | "据核查" | 明确核查方式和来源 |
| 格式不一 | 有的有编号有的没有 | 统一添加编号 |
| 页码缺失 | 未标注关键页码 | 补充页码信息 |
| 附件缺失 | 正文提到但无附件 | 补充附件或删去引用 |
```

### 5. 引用模板库

提供常用引用模板：

#### 5.1 财务数据引用
```markdown
根据 [会计师事务所] 出具的《审计报告》（[报告号]），截至 [日期]，发行人 [财务指标] 为 [数值]。（见附件 X-X）
```

#### 5.2 业务数据引用
```markdown
根据 [来源] 统计/数据显示，[数据内容]。根据对 [访谈对象] 的访谈（访谈纪要 [编号]），[补充说明]。（见附件 X-X）
```

#### 5.3 法律事项引用
```markdown
根据 [律师事务所] 出具的《法律意见书》（[文号]），[法律结论]。（见附件 X-X）
根据 [政府部门] 出具的 [证明名称]（[日期]），[证明内容]。（见附件 X-X）
```

#### 5.4 核查程序引用
```markdown
保荐人履行了以下核查程序：
1. [程序 1]，详见 [核查记录编号]；
2. [程序 2]，详见 [核查记录编号]；
3. [程序 3]，详见 [核查记录编号]。
```

## 注意事项

- 引用应准确、完整、可追溯
- 格式应统一规范
- 涉密信息应脱敏
- 附件应清晰可读
- 引用材料应在有效期内
- 注意版权问题
