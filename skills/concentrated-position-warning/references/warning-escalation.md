# 预警升级机制

## 一、预警等级

| 等级 | 颜色 | 定义 | 响应要求 |
|------|------|------|----------|
| 黄色预警 | 黄色 | 集中度偏高 | 发送提示 |
| 橙色预警 | 橙色 | 集中度过高 | 要求改善 |
| 红色预警 | 红色 | 集中度极高 | 强制干预 |

## 二、升级条件

### 2.1 黄色→橙色
- 集中度指标超过预警线
- 持续 3 个交易日未改善
- 客户拒绝配合

### 2.2 橙色→红色
- 集中度指标超过警戒线
- 持续 2 个交易日未改善
- 证券价格大幅波动

### 2.3 降级条件
- 集中度指标降至下一级以下
- 持续 2 个交易日稳定
- 风险得到有效控制

## 三、升级流程

### 3.1 自动升级
```
系统检测 → 触发条件 → 自动升级 → 发送通知 → 记录日志
```

### 3.2 人工升级
```
监控人员发现 → 风险评估 → 负责人审批 → 执行升级 → 通知客户
```

### 3.3 紧急升级
```
重大风险事件 → 直接升级 → 立即通知 → 启动应急预案
```

## 四、响应措施

### 4.1 黄色预警响应
- 发送短信/微信提示
- 客户经理知晓
- 记录预警信息
- 持续监控

### 4.2 橙色预警响应
- 发送正式通知
- 客户经理电话沟通
- 要求制定改善计划
- 每日监控变化
- 限制新开仓

### 4.3 红色预警响应
- 发送紧急通知
- 负责人联系客户
- 要求立即改善
- 准备强制措施
- 上报风控部门
- 实时监控

## 五、升级通知模板

### 5.1 黄色预警通知
```
【XX 证券】持仓集中度提示

尊敬的 [客户]：

您的账户持仓集中度有所上升，
最大持仓占比 X%。

建议您适当分散持仓，防范风险。
```

### 5.2 橙色预警通知
```
【XX 证券】持仓集中度预警

尊敬的 [客户]：

您的账户持仓集中度较高，
最大持仓占比 X%，已超过预警线。

请您尽快调整持仓结构。
```

### 5.3 红色预警通知
```
【XX 证券】紧急预警

尊敬的 [客户]：

您的账户持仓集中度过高，
存在较大风险。

请您立即调整持仓，
否则我司将采取限制措施。
```

## 六、升级记录

| 字段 | 说明 |
|------|------|
| 客户 ID | 客户标识 |
| 原等级 | 升级前等级 |
| 新等级 | 升级后等级 |
| 升级时间 | 升级发生时间 |
| 升级原因 | 触发升级的原因 |
| 操作人员 | 执行升级的人员 |
| 通知情况 | 通知客户情况 |

## 七、升级评估

### 7.1 定期评估
- 每周评估升级情况
- 分析升级原因
- 评估措施效果
- 优化升级机制

### 7.2 特殊情况
- 市场剧烈波动时暂停升级
- 客户特殊情况可申请暂缓
- 重大节假日前审慎升级
