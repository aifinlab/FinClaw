# 行业数据要求说明

本技能需要行业数据支持较强，主要体现在以下方面：

## 一、为什么需要行业数据
仅凭内部交易异常，往往只能发现“看起来不寻常”的模式，难以判断其是否真正偏离行业常态。

例如：
- 某批发贸易企业的高频大额收付款，在其行业内可能是正常现象；
- 某零售个人客户的多笔小额收款，可能与真实经营活动有关；
- 某地区特定时段的集中交易，可能与季节性或区域结算习惯有关。

## 二、优先需要的行业数据
1. 同行业客户交易频率区间
2. 同行业客户单笔金额与月均流水分布
3. 行业常见对手方集中度
4. 行业季节性、账期性与结算特征
5. 高风险行业特征库
6. 高风险国家/地区、渠道、名单信息

## 三、缺失时的降级要求
若缺少上述数据，只能输出：
- 内部交易异常线索
- 待复核名单
- 需要补充的外部数据清单

不得输出：
- 明确的同业显著偏离结论
- 高置信度洗钱性质判断
- 依赖外部名单的命中描述
