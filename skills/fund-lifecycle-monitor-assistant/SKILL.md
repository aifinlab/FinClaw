# 技能说明

本技能用于支持基金业务中的产品监测工作，重点解决「根据规模、申赎、业绩、渠道热度和客户反馈判断基金产品所处生命周期并输出监测提示。」这类任务。输出默认使用中文，要求结构清楚、证据边界明确，并能被投研、产品、销售、投顾或合规团队直接复用。

# 本任务重点关注的问题

- 产品处在哪个阶段
- 当前最主要的经营矛盾是什么
- 阶段变化是短期波动还是趋势切换
- 下一阶段最该盯住哪些指标

# 适用范围

- 募集期、成长期、成熟期和维护期判断
- 规模与申赎趋势监测
- 阶段性风险和动作建议
- 产品运营例会监控

# 不适用范围

- 缺少阶段性运营数据却要求精细判断
- 把生命周期结论写成外部承诺
- 忽略宏观市场和渠道环境影响

# 输入要求

优先输入包括：

- 规模、份额与申赎数据
- 净值和回撤表现
- 渠道热度或销售数据
- 阶段目标与运营策略

可选增强输入包括：

- 客户反馈
- 竞品动态
- 营销活动记录

当输入不完整时，必须先列出缺失材料，再基于现有信息输出阶段性判断，不得编造未确认事实。

# 核心分析框架

## 一、先判断产品角色和所处阶段

- 明确产品定位、目标客群、渠道场景和所处生命周期。
- 区分内部管理讨论和对外表达口径。
- 看清产品在线路中的分工，而不是只看单一指标。

## 二、围绕产品事实和协同问题展开

- 整理正式披露材料、内部口径、渠道反馈和运营数据。
- 识别定位、标签、材料、同类竞争和生命周期中的关键矛盾。
- 先找到真正影响经营和沟通的问题。

## 三、输出能落地的判断

- 建议要带动作和优先级。
- 区分短期修补、中期优化和长期定位问题。
- 对外可见内容要保留复核边界。

# 推荐工作流

1. 先明确产品所处阶段、目标客群、定位约束和业务目标。
2. 整理产品资料、同类对照、渠道反馈和运营数据，建立统一口径。
3. 识别产品定位、生命周期、标签、材料或运维流程中的关键问题。
4. 给出可执行的优化建议、信息补充清单和后续动作。
5. 输出时同时保留适用边界，避免把内部分析包装成外部承诺。

# 关键判断视角

- 产品定位、客群与渠道适配度
- 同类差异化、阶段性表现与运营状态
- 材料、标签、卖点和内部口径的一致性
- 待补充数据、流程依赖和落地优先级

# 输出要求

建议输出至少覆盖以下模块：

- 生命周期阶段判断
- 关键监测指标
- 异常信号
- 动作建议
- 待补充数据

若任务涉及对客材料、话术或解释文案，必须同时输出「不能直接对外使用的前提说明」或「需正式复核」的提醒。

# 输出约束

- 不得制造超出合同和正式披露的新卖点。
- 不得把内部讨论口径直接转成外部宣传。
- 不得跳过定位冲突和渠道适配问题。

# 质量检查清单

- 是否先界定了任务边界、时间窗口、样本范围和使用场景。
- 是否把已确认事实、解释性判断和待核验事项分开表达。
- 是否给出了证据链，而不是只写结论标签。
- 是否保留了风险提示、限制说明和后续动作。
- 若内容可能进入对客或外部场景，是否保留了复核边界。
- 当前 skill 特别要盯住的判断视角：
- 产品定位、客群与渠道适配度
- 同类差异化、阶段性表现与运营状态
- 材料、标签、卖点和内部口径的一致性
- 待补充数据、流程依赖和落地优先级
