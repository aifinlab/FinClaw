# 客户产品渗透分析助手

## 1. 业务问题

银行零售业务收入和客户粘性与产品渗透深度高度相关。多产品客户流失率低、贡献度高，但实际经营中大量客户仅持有单一产品，交叉销售潜力未被充分挖掘。需要系统分析产品渗透率和持有结构，识别交叉销售机会和单产品依赖风险。

## 2. 适用场景

- 客群产品渗透率现状分析与对标
- 识别高价值但产品持有单一的客户
- 营销活动前筛选交叉销售目标客户
- 产品到期续接策略制定
- 分行/支行产品渗透率横向对比
- 新产品上线后渗透率追踪

## 3. 输入

| 输入项 | 说明 | 是否必填 |
|--------|------|----------|
| 客户列表 | 待分析的客户编号集合 | 必填 |
| 产品持有明细 | 每位客户当前持有的产品清单 | 必填 |
| 产品分类标准 | 产品归类规则（默认五大类） | 选填 |
| 客户价值层级 | 分层结果（如有） | 选填 |
| 产品到期信息 | 理财/存款/贷款到期日 | 选填 |
| 历史持有记录 | 产品持有变化历史（如有） | 选填 |
| 机构维度 | 分行/支行/网点归属 | 选填 |

## 4. 输出

- **渗透率统计**：各产品类别在目标客群中的渗透率
- **持有结构分布**：客户按持有产品数量分组的占比分布
- **交叉销售机会**：高价值但低渗透客户清单及推荐产品
- **单产品风险客户**：仅持有单一产品的客户识别
- **产品组合分析**：常见产品组合模式及关联关系
- **到期提醒**：近期产品到期客户及续接建议

## 5. 默认分析维度

- **存款类**：活期存款、定期存款、结构性存款、大额存单
- **理财产品类**：银行理财、代销基金、代销保险、贵金属
- **贷款类**：住房贷款、消费贷款、经营贷款、信用卡
- **电子渠道类**：手机银行、网上银行、微信银行
- **其他服务类**：代发工资、代扣代缴、安全工具（U盾/短信验证）

## 6. 默认分析框架

1. **产品归类**：将客户持有产品按五大类归类汇总
2. **渗透率计算**：调用 bank-calc-utils 计算各产品类别渗透率
3. **持有深度分析**：统计客户持有产品类别数分布（1类/2类/3类/4类/5类）
4. **交叉销售识别**：
   - 高价值 + 持有类别数<=2 → 优先交叉销售对象
   - 分析常见产品组合，推荐缺失产品
5. **风险识别**：标记仅持有单一产品的客户为粘性风险客户
6. **到期机会**：汇总近30/60/90天到期产品，生成续接机会清单
7. **结论输出**：先给出整体渗透率结论和关键发现，再展示明细

## 7. 安全边界

- 不编造数据；不夸大结论；缺失信息标注"未获取"或"待核实"
- 不替代正式审批、正式风控、正式合规、正式报送、正式处置、正式责任认定、正式管理决策
- 不替代营销准入判断、授信审批决策、客户适当性评估
- 产品推荐仅基于渗透率分析，不构成客户适当性评估
- 交叉销售建议需结合客户风险偏好和适当性要求后方可执行

## 8. 上游依赖

- **bank-calc-utils**：渗透率计算、分布统计、占比分析等数值运算

## 9. 可联动下游 Skills
