# 客户经营机会清单生成助手

## 1. 业务问题

银行各类客户分析产出分散在不同系统和报告中，客户经理难以快速获取汇总后的可执行经营动作。需要将客户分层、流失预警、产品渗透、沉睡资金激活、提前还款挽留等多维分析结果整合为一份优先级排序的机会清单，直接对接客户经理日常经营工作。

## 2. 适用场景

- 客户经理每日/每周经营任务清单生成
- 营销活动前目标客户筛选与动作分配
- 管理层查看客户经营机会全景
- 团队任务分配与工作量平衡
- 经营动作执行效果追踪与复盘
- 季末/年末客户经营冲刺重点梳理

## 3. 输入

| 输入项 | 说明 | 是否必填 |
|--------|------|----------|
| 客户分层结果 | 来自 customer-segmentation 的分层数据 | 选填 |
| 流失预警结果 | 来自 customer-churn-alert 的预警清单 | 选填 |
| 产品渗透分析 | 来自 customer-product-penetration 的交叉销售机会 | 选填 |
| 沉睡资金分析 | 来自 dormant-account-analysis 的激活机会 | 选填 |
| 提前还款分析 | 来自 early-repayment-churn-analysis 的挽留对象 | 选填 |
| 客户经理归属 | 客户与客户经理的对应关系 | 选填 |
| 优先级规则 | 自定义优先级权重（默认使用内置规则） | 选填 |

备注：至少需要一项上游分析结果作为输入。

## 4. 输出

- **机会清单总表**：按优先级排序的客户经营机会列表
- **机会分类汇总**：按机会类型（挽留/提升/激活/交叉销售/续接）分组统计
- **客户经理视图**：按客户经理分组的任务清单
- **每条机会包含**：客户编号、客户名称、机会类型、优先级、建议动作、来源分析、关键数据
- **执行建议**：每条机会的推荐跟进方式和时间窗口

## 5. 默认分析维度

- 机会类型：流失挽留、价值提升、沉睡激活、交叉销售、到期续接、提前还款挽留
- 客户价值层级：高价值客户机会优先
- 紧迫程度：高风险流失、即将到期产品优先
- 预估收益：AUM提升潜力、中收贡献潜力
- 执行难度：联系方式完整度、历史互动响应率
- 客户经理维度：按管户归属分配

## 6. 默认分析框架

1. **数据汇集**：接收各上游技能输出的分析结果
2. **机会识别**：将每条分析结论转化为一条可执行机会
3. **优先级评分**：综合评估每条机会的优先级
   - 客户价值权重（40%）：高价值客户机会得分更高
   - 紧迫程度权重（30%）：高风险流失、近期到期优先
   - 预估收益权重（20%）：预期贡献度提升幅度
   - 执行可行性权重（10%）：联系渠道可达性
4. **去重合并**：同一客户多条机会合并为综合经营建议
5. **清单生成**：按优先级排序输出，附带建议动作和时间窗口
6. **分组输出**：按客户经理、机会类型等维度分组展示

## 7. 安全边界

- 不编造数据；不夸大结论；缺失信息标注"未获取"或"待核实"
- 不替代正式审批、正式风控、正式合规、正式报送、正式处置、正式责任认定、正式管理决策
- 不替代营销准入判断、授信审批决策、客户适当性评估
- 机会清单仅作为经营参考，具体营销动作需遵守合规要求
- 优先级排序为建议性质，客户经理可根据实际情况调整

## 8. 上游依赖

- **customer-segmentation**：客户价值分层结果
- **customer-churn-alert**：流失预警客户清单
- **customer-product-penetration**：交叉销售机会
- **dormant-account-analysis**：沉睡资金激活机会
- **early-repayment-churn-analysis**：提前还款挽留对象
