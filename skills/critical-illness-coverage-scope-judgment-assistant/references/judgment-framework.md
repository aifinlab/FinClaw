# 责任触发条件框架

## 1. 明确责任模块

- 先确认案件适用的是单次重疾、多次赔付重疾、特定疾病责任、癌症责任或其他附加给付责任。
- 若责任模块不清，先写明“责任模块待确认”，不要直接代入标准重疾责任。

## 2. 判断是否属于合同保障范围

- 当前疾病或状态是否属于合同列明的保障对象。
- 定义命中后，是否仍受年龄、分组、次数、间隔期或首次确诊要求限制。

## 3. 判断是否满足启动条件

- 是否在保险期间内发生。
- 是否满足等待期及首次确诊要求。
- 是否满足疾病定义、状态条件、治疗条件或结果条件。
- 是否有足够事实和材料支持上述条件。

## 4. 判断当前结论成熟度

- 条款清晰、定义清晰、材料充分时，可给出较明确的初步判断。
- 条款不全、定义不清、病理待回报、分期待明确、关键材料缺失或事实冲突时，优先写“暂无法判断”或“需补充核实”。
