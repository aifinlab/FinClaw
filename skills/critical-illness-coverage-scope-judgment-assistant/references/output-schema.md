# 默认输出结构

除非用户另有要求，按以下顺序输出：

## 一、案件基本信息

- 案件号
- 产品/险种
- 疾病名称
- 首次就诊/确诊时间
- 当前案件状态
- 材料来源或记录类型

## 二、责任判断摘要

- 用 3 到 5 条简明要点概括核心结论
- 优先说明是否初步可能属于责任范围、主要支持因素、主要限制因素和当前判断边界
- 如存在材料缺失、条款不清、定义匹配不足或争议较大，要在摘要中提示

## 三、重疾事实整理

- 发病或发现经过
- 首次就诊情况
- 确诊时间
- 诊断名称
- 病理、影像或实验室依据
- 手术或治疗情况
- 已提交材料情况
- 其他与责任判断相关的重要事实

## 四、疾病定义匹配分析

- 当前案件对应的重疾定义名称
- 条款定义中的核心判断条件
- 当前事实与条款定义的匹配情况
- 已满足的定义条件
- 尚未明确或尚未满足的定义条件
- 如无法确认，应说明缺失的关键判断条件

## 五、责任触发条件分析

- 当前案件对应的责任模块
- 重疾责任是否属于合同保障范围
- 是否满足责任启动条件
- 触发判断所依据的事实和条款点
- 如无法确认，应说明缺失的关键判断条件

## 六、免责与争议点分析

- 是否可能涉及等待期
- 是否可能涉及既往症或投保前已存在症状、检查异常、相关诊疗记录
- 是否可能涉及责任免除情形
- 是否可能涉及轻症、中症、重疾定义边界争议
- 是否存在事实不清、材料不足、表述冲突或条款解释空间

## 七、材料支持度判断

- 当前材料是否足以支持初步判断
- 已具备的关键材料
- 缺失或待补充的关键材料
- 哪些材料缺失会直接影响责任结论

## 八、初步判断结论

仅使用以下之一：

- 可能属于责任范围
- 可能部分匹配但需进一步核验
- 可能不属于责任范围
- 暂无法判断

同时补充：

- 主要支持依据
- 主要限制依据
- 当前结论是否需要进一步复核

## 九、后续处理建议

- 是否建议补充材料
- 是否建议转人工复核
- 是否建议结合完整条款再次核验
- 是否建议进一步核实确诊时间、病理结论、影像结论、疾病严重程度或等待期等关键事项
- 是否存在客户沟通风险或争议升级风险
